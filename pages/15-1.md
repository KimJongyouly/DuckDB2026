# 15-1. Extension 설치

확장을 사용하려면 먼저 설치(INSTALL)한 뒤 세션마다 로드(LOAD)해야 한다. 설치는 한 번만 하면 되고, 로드는 DuckDB를 새로 시작할 때마다 필요하다. `autoinstall`·`autoload` 설정을 켜두면 이 과정을 자동화할 수 있다.

### 확장 관리 기본

```sql
-- 확장 설치 (DuckDB 공식 레포에서 다운로드)
INSTALL httpfs;
INSTALL spatial;
INSTALL fts;
INSTALL excel;
INSTALL iceberg;
INSTALL delta;
INSTALL json;
INSTALL parquet;    -- 기본 내장, 별도 설치 불필요

-- 설치 후 로드 (세션마다 필요)
LOAD httpfs;
LOAD spatial;

-- 한 번에 설치 + 로드
INSTALL spatial; LOAD spatial;

-- 자동 설치 · 자동 로드 설정 (권장)
SET autoinstall_known_extensions = true;
SET autoload_known_extensions = true;
-- 이후 알려진 확장은 사용 시 자동 설치·로드
```

### 확장 목록 조회

```sql
-- 설치된 확장 전체 목록
SELECT *
FROM duckdb_extensions()
ORDER BY extension_name;

-- 컬럼 의미
-- extension_name    : 확장 이름
-- loaded            : 현재 로드 여부
-- installed         : 설치 여부
-- install_path      : 설치 경로
-- description       : 설명
-- aliases           : 별칭
-- extension_version : 버전

-- 로드된 확장만
SELECT extension_name, extension_version
FROM duckdb_extensions()
WHERE loaded = true;

-- 설치 가능한 공식 확장 목록
SELECT extension_name, description
FROM duckdb_extensions()
ORDER BY extension_name;
```

### 확장 설치 경로와 버전 관리

```sql
-- 설치 경로 확인
SELECT install_path FROM duckdb_extensions()
WHERE extension_name = 'spatial';
-- ~/.duckdb/extensions/v1.1.0/linux_amd64/spatial.duckdb_extension

-- 특정 버전 설치
INSTALL spatial FROM 'http://extensions.duckdb.org/v1.0.0';

-- 확장 업데이트
UPDATE EXTENSIONS;
UPDATE EXTENSIONS (spatial);   -- 특정 확장만 업데이트

-- 오프라인 환경: 로컬 파일에서 로드
LOAD '/path/to/my_extension.duckdb_extension';

-- 서명되지 않은 확장 허용 (개발·테스트용)
SET allow_unsigned_extensions = true;
LOAD '/path/to/unsigned.duckdb_extension';
```

### ~/.duckdbrc에 영구 설정

```sql
-- ~/.duckdbrc
SET autoinstall_known_extensions = true;
SET autoload_known_extensions = true;

-- 자주 쓰는 확장 미리 로드
LOAD httpfs;
LOAD spatial;
LOAD json;
```

> **팁:** `~/.duckdbrc` 파일에 `autoinstall_known_extensions = true`를 설정해 두면, DuckDB CLI를 시작할 때마다 별도 명령 없이 공식 확장이 자동으로 설치·로드된다. 개발 환경에서 특히 편리하다.

확장 설치 방법을 이해했다면, 이제 가장 많이 쓰이는 확장부터 하나씩 살펴보자. 첫 번째는 원격 파일을 로컬처럼 쿼리할 수 있게 해주는 `httpfs`다.
