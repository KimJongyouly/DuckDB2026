# 05-2. 문자열 함수

텍스트 데이터는 어느 데이터셋에나 존재한다. 이름, 주소, 로그 메시지, URL 등 문자열을 정제하고 분석하는 작업은 ETL 파이프라인과 탐색적 분석 모두에서 빈번하게 등장한다. DuckDB의 문자열 함수는 표준 SQL 함수부터 정규 표현식, 유사도 측정까지 폭넓게 제공한다.

### 기본 조작

```sql
-- 길이
SELECT LENGTH('DuckDB');           -- 6 (문자 수)
SELECT OCTET_LENGTH('DuckDB');     -- 6 (바이트 수, UTF-8에서 한글은 3)
SELECT BIT_LENGTH('AB');           -- 16

-- 대소문자
SELECT UPPER('hello');    -- 'HELLO'
SELECT LOWER('WORLD');    -- 'world'
SELECT INITCAP('hello world');  -- 'Hello World'

-- 공백 처리
SELECT TRIM('  hello  ');        -- 'hello'
SELECT LTRIM('  hello  ');       -- 'hello  '
SELECT RTRIM('  hello  ');       -- '  hello'
SELECT TRIM('xxhelloxx', 'x');   -- 'hello' (특정 문자 제거)

-- 패딩
SELECT LPAD('42', 6, '0');    -- '000042'
SELECT RPAD('ok', 5, '.');    -- 'ok...'
SELECT REPEAT('ab', 3);       -- 'ababab'

-- 역순
SELECT REVERSE('DuckDB');     -- 'BDkcuD'
```

### 검색과 추출

```sql
-- 위치 찾기
SELECT POSITION('DB' IN 'DuckDB');           -- 5
SELECT STRPOS('DuckDB', 'DB');               -- 5 (동일)
SELECT INSTR('DuckDB', 'ck');               -- 3

-- 부분 문자열
SELECT SUBSTRING('DuckDB', 1, 4);           -- 'Duck'
SELECT SUBSTR('DuckDB', 5);                 -- 'DB' (끝까지)
SELECT SUBSTRING('DuckDB', -2);             -- 'DB' (뒤에서 2번째부터)
SELECT LEFT('DuckDB', 4);                   -- 'Duck'
SELECT RIGHT('DuckDB', 2);                  -- 'DB'

-- 포함 여부
SELECT 'DuckDB' LIKE '%Duck%';              -- true
SELECT 'duckdb' ILIKE '%DUCK%';             -- true (대소문자 무시)
SELECT CONTAINS('DuckDB', 'uck');           -- true
SELECT PREFIX('DuckDB', 'Duck');            -- true (starts_with)
SELECT SUFFIX('DuckDB', 'DB');              -- true (ends_with)
SELECT STARTS_WITH('DuckDB', 'Duck');       -- true
SELECT ENDS_WITH('DuckDB', 'DB');           -- true
```

> **팁:** `CONTAINS`, `PREFIX`, `SUFFIX`는 `LIKE '%...%'`보다 의도가 명확하고 짧다. 또한 `ILIKE`는 대소문자를 구분하지 않으므로 사용자 입력 검색에 유용하다.

### 변환과 포맷

```sql
-- 치환과 분리
SELECT REPLACE('hello world', 'world', 'DuckDB');  -- 'hello DuckDB'
SELECT TRANSLATE('hello', 'aeiou', '*****');        -- 'h*ll*'
SELECT STRING_SPLIT('a,b,c', ',');                  -- ['a', 'b', 'c']
SELECT STRING_SPLIT_REGEX('one  two   three', '\s+'); -- ['one', 'two', 'three']
SELECT SPLIT_PART('a,b,c', ',', 2);                -- 'b'

-- 연결
SELECT 'Duck' || 'DB';                             -- 'DuckDB'
SELECT CONCAT('Duck', 'DB', '!');                  -- 'DuckDB!'
SELECT CONCAT_WS(', ', 'Seoul', 'Busan', 'Incheon'); -- 'Seoul, Busan, Incheon'

-- 포맷팅
SELECT FORMAT('{} has {} items', 'cart', 5);       -- 'cart has 5 items'
SELECT PRINTF('%05d', 42);                          -- '00042'
SELECT PRINTF('%.2f%%', 98.765);                   -- '98.77%'
```

### 정규 표현식

고정된 패턴으로 처리하기 어려운 자유 형식 텍스트에서 특정 구조를 추출하거나 검증할 때 정규 표현식을 사용한다. DuckDB는 RE2 엔진을 내장해, 별도 UDF 없이 SQL 안에서 바로 정규식을 실행한다.

```sql
-- 매칭 여부
SELECT REGEXP_MATCHES('user@email.com', '^[\w.]+@[\w.]+\.\w+$');  -- true

-- 추출 (첫 번째 매칭)
SELECT REGEXP_EXTRACT('price: $42.50', '\d+\.\d+');   -- '42.50'

-- 전체 추출 (캡처 그룹)
SELECT REGEXP_EXTRACT('2024-03-15', '(\d{4})-(\d{2})-(\d{2})', 1); -- '2024'
SELECT REGEXP_EXTRACT_ALL('cat bat hat', '[a-z]at');  -- ['cat', 'bat', 'hat']

-- 치환
SELECT REGEXP_REPLACE('  extra   spaces  ', '\s+', ' ');  -- ' extra spaces '
SELECT REGEXP_REPLACE('aabbcc', '(.)\1', '\1', 'g');      -- 'abc' (중복 제거)

-- 분리
SELECT REGEXP_SPLIT_TO_ARRAY('one1two2three', '\d');  -- ['one', 'two', 'three']
```

> **주의:** DuckDB의 정규식 엔진은 RE2를 사용하므로 `(?=...)` 같은 lookahead 문법을 지원하지 않는다. PCRE 문법에 익숙하다면 일부 패턴을 RE2 호환 방식으로 재작성해야 한다.

### 유사도와 변환

오타가 섞인 이름을 매칭하거나, 두 문자열이 얼마나 비슷한지 수치로 비교해야 할 때 유사도 함수를 사용한다. 데이터 클렌징이나 퍼지 조인 작업에서 특히 유용하다.

```sql
-- 문자열 유사도 (0~1)
SELECT JARO_SIMILARITY('kitten', 'sitting');          -- 0.746...
SELECT JARO_WINKLER_SIMILARITY('kitten', 'sitting');  -- 0.746...

-- 편집 거리 (Levenshtein)
SELECT LEVENSHTEIN('kitten', 'sitting');  -- 3

-- 인코딩
SELECT TO_BASE64('Hello');         -- 'SGVsbG8='
SELECT FROM_BASE64('SGVsbG8=');    -- 'Hello'
SELECT MD5('DuckDB');              -- 16진수 해시
SELECT SHA256('DuckDB');           -- SHA-256 해시

-- ASCII / CHR
SELECT ASCII('A');   -- 65
SELECT CHR(65);      -- 'A'
SELECT ORD('가');    -- 44032 (유니코드 코드포인트)
```

문자열을 정제했다면, 이제 분석에서 또 다른 핵심 축인 날짜와 시간을 다뤄보자.
