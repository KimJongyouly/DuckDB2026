# 02-1. 커넥션 개념 이해

DuckDB의 연결은 매우 가볍다. 네트워크 소켓을 열거나 인증을 거칠 필요 없이, 라이브러리 함수 호출 한 번으로 연결이 만들어진다. 연결을 생성할 때 결정해야 할 핵심 사항은 딱 하나다.

> **데이터를 디스크에 저장할 것인가, 메모리에만 둘 것인가?**

이 결정에 따라 연결 방식이 달라진다. 처음 DuckDB를 접한다면 "어떤 모드를 골라야 할까?"라는 질문이 생기는 게 당연하다. 간단히 말하면, 분석 결과를 나중에도 다시 쓸 필요가 있으면 영구 모드, 지금 이 순간 탐색이 목적이라면 인메모리 모드가 적합하다.

```
┌───────────────────────────────────────┐
│           DuckDB 연결 방식              │
├─────────────────────┬─────────────────┤
│   영구 모드          │  인메모리 모드   │
│ duckdb('file.db')   │  duckdb()       │
│                     │  duckdb(':memory:') │
│ 데이터가 파일에 저장  │ 프로세스 종료시  │
│                     │ 데이터 소멸      │
└─────────────────────┴─────────────────┘
```

연결을 생성하는 문법은 사용하는 클라이언트 API마다 조금씩 다르지만, 핵심 개념은 동일하다. 아래 예제에서는 CLI와 Python을 기준으로 설명한다.

두 모드의 차이를 먼저 그림으로 파악하고, 이어서 각 모드를 구체적으로 살펴보겠다.
