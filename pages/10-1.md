# 10-1. 와일드카드로 여러 파일 읽기

데이터가 날짜별·지역별로 수십, 수백 개 파일에 흩어져 있을 때 파일을 하나씩 지정하면 쿼리가 길어지고 유지보수가 어려워진다. DuckDB는 파일 경로에 글로브(glob) 패턴을 직접 사용할 수 있어, 패턴 하나로 일치하는 모든 파일을 단일 테이블처럼 읽는다.

### 기본 와일드카드 패턴

```sql
-- * : 같은 디렉터리의 모든 파일
SELECT * FROM 'data/*.parquet';
SELECT * FROM 'logs/*.csv';
SELECT * FROM 'events/*.json';

-- ? : 임의의 한 문자
SELECT * FROM 'data/202?_sales.parquet';   -- 2020~2029

-- ** : 하위 디렉터리 재귀 탐색
SELECT * FROM 'warehouse/**/*.parquet';
SELECT * FROM 'data/**/*.csv';

-- 패턴 조합
SELECT * FROM 'logs/2024_0[1-6]_*.csv';   -- 2024년 1~6월
```

### 파일 형식별 다중 파일 읽기

```sql
-- CSV
SELECT * FROM read_csv('data/sales_*.csv', header=true);

-- Parquet
SELECT * FROM read_parquet('data/*.parquet');

-- JSON (NDJSON)
SELECT * FROM read_json('logs/*.ndjson', format='newline_delimited');

-- 혼합 형식은 불가 (같은 형식끼리만)
-- 형식이 다르면 UNION ALL로 결합
SELECT * FROM 'jan.csv'
UNION ALL
SELECT * FROM 'jan.parquet';  -- 동일 스키마라면 가능
```

### 배열로 파일 목록 지정

```sql
-- 파일 목록 배열
SELECT * FROM read_parquet([
    'data/2024_01.parquet',
    'data/2024_02.parquet',
    'data/2024_03.parquet'
]);

-- CSV 배열
SELECT * FROM read_csv([
    'region/seoul.csv',
    'region/busan.csv',
    'region/incheon.csv'
], header=true);

-- 동적 파일 목록 (Python에서 생성)
```

```python
import duckdb
from pathlib import Path

con = duckdb.connect()

# 특정 조건의 파일만 선택
files = sorted(Path('data').glob('2024_*.parquet'))
file_list = [str(f) for f in files]

# 배열로 전달
con.execute(f"""
    SELECT * FROM read_parquet({file_list})
""").df()

# 또는 glob으로 정렬 보장
import glob
files = sorted(glob.glob('data/2024_*.parquet'))
```

### 파일 출처 추적 (filename 옵션)

여러 파일을 한 번에 읽으면 각 행이 어느 파일에서 왔는지 알기 어렵다. `filename=true` 옵션을 지정하면 DuckDB가 각 행에 원본 파일 경로를 담은 `filename` 열을 자동으로 추가한다.

> **팁:** `filename=true`는 데이터 품질 검사나 파이프라인 디버깅 시 특히 유용하다. 파일 이름 자체에 날짜·지역 정보가 인코딩된 경우 `REGEXP_EXTRACT`로 파티션 정보를 직접 파싱할 수 있다.

```sql
-- 파일 이름 열 추가
SELECT filename, * FROM read_parquet('data/*.parquet', filename=true);
-- filename              | id | date       | amount
-- ──────────────────────┼────┼────────────┼───────
-- data/2024_01.parquet  |  1 | 2024-01-05 | 15000
-- data/2024_01.parquet  |  2 | 2024-01-12 |  8500
-- data/2024_02.parquet  | 51 | 2024-02-03 | 22000

-- 파일별 집계
SELECT
    filename,
    COUNT(*) AS row_count,
    SUM(amount) AS total,
    MIN(date) AS earliest,
    MAX(date) AS latest
FROM read_parquet('data/*.parquet', filename=true)
GROUP BY filename
ORDER BY filename;

-- 파일 이름에서 날짜 파싱
SELECT
    REGEXP_EXTRACT(filename, '(\d{4}_\d{2})') AS month,
    COUNT(*) AS rows,
    SUM(amount) AS total
FROM read_parquet('data/*.parquet', filename=true)
GROUP BY month
ORDER BY month;
```

> **팁:** `file_row_number=true`는 CSV 파일에서 특정 행에 파싱 오류가 발생했을 때 원본 파일의 정확한 위치를 찾는 데 유용하다. Parquet에는 지원되지 않으며 CSV 전용 옵션이다.

### 행 번호 추적 (file_row_number 옵션)

```sql
-- 파일 내 행 번호 추적 (디버깅, 데이터 품질 검사에 유용)
SELECT file_row_number, filename, *
FROM read_csv(
    'data/*.csv',
    filename=true,
    file_row_number=true,
    header=true
)
WHERE amount < 0;  -- 음수 금액 행의 파일과 위치 확인
-- file_row_number | filename   | id | amount
-- ────────────────┼────────────┼────┼───────
-- 142             | jan.csv    | 88 | -500
-- 7               | feb.csv    | 212| -200
```

### 파일 탐색과 필터링

```sql
-- 파일 목록 조회 (glob 함수)
SELECT * FROM glob('data/*.parquet');
-- file
-- ────────────────────
-- data/2024_01.parquet
-- data/2024_02.parquet
-- data/2024_03.parquet

-- 파일 크기, 수정 시각 포함
SELECT * FROM glob('data/**/*.parquet');
-- file | size | last_modified
-- ─────┼──────┼──────────────

-- 조건부 파일 읽기 (Python에서 필터)
```

```python
import duckdb
from pathlib import Path
from datetime import datetime, timedelta

con = duckdb.connect()

# 최근 7일 파일만 읽기
cutoff = datetime.now() - timedelta(days=7)
recent_files = [
    str(f) for f in Path('data').glob('*.parquet')
    if datetime.fromtimestamp(f.stat().st_mtime) > cutoff
]

if recent_files:
    df = con.execute(f"SELECT * FROM read_parquet({recent_files})").df()
```
