# 00-3. 이 책의 구성과 독자 대상

### 누구를 위한 책인가

이 책은 다음과 같은 독자를 염두에 두고 쓰였다.

**데이터 분석가·데이터 사이언티스트:** SQL은 알지만 대용량 데이터를 다룰 때 Pandas가 느려지거나 메모리 오류가 나는 상황을 경험해본 사람. DuckDB가 어떻게 그 갭을 채우는지 실용적으로 배우고 싶은 독자.

**백엔드·데이터 엔지니어:** 데이터 파이프라인에 경량 분석 레이어를 추가하거나, 로컬 데이터 레이크를 구축하고 싶은 엔지니어. Parquet, Iceberg, Delta Lake 같은 포맷과 DuckDB를 연결하는 방법을 익히고 싶은 독자.

**SQL을 배우고 싶은 개발자:** 서버 설치 없이 바로 SQL을 실습할 수 있는 환경이 필요한 사람. DuckDB의 CLI는 SQLite보다 훨씬 풍부한 SQL 기능을 제공하면서도 설치가 단순하다.

SQL에 대한 기본 이해(SELECT, WHERE, GROUP BY 정도)가 있으면 충분하다. Python 코드가 등장하는 장에서는 Python 기초 지식이 도움이 되지만 필수는 아니다.

> **팁:** 이 책을 읽는 가장 좋은 방법은 각 장의 예제를 직접 실행해보는 것이다. DuckDB CLI나 Python 환경을 옆에 열어두고, 코드를 직접 따라 치면서 결과를 확인하면 훨씬 빠르게 익힐 수 있다.

### 이 책의 구성

이 책은 총 7부로 이루어져 있다.

**1부 (1~2장)** 는 설치부터 첫 연결까지를 다룬다. DuckDB를 내 환경에 설치하고, 파일 DB와 인메모리 DB의 차이를 이해하며, 처음 쿼리를 실행하는 것을 목표로 한다.

**2부 (3~6장)** 는 SQL의 핵심을 DuckDB 관점에서 정리한다. 기본 쿼리 문법부터 윈도우 함수, 풍부한 데이터 타입, 수백 가지 내장 함수까지 다룬다.

**3부 (7~11장)** 는 데이터 입출력의 실전이다. CSV, JSON, Parquet을 읽고 쓰는 법, 수백 개의 파일을 한 번에 쿼리하는 법, Apache Iceberg와 Delta Lake 연동까지 다룬다.

**4부 (12~14장)** 는 언어별 클라이언트 API를 정리한다. Python과의 깊은 통합을 중심으로, Java, Node.js, WebAssembly까지 폭넓게 살펴본다.

**5부 (15~16장)** 는 확장(Extensions)을 다룬다. httpfs로 S3를 직접 쿼리하고, Spatial 확장으로 공간 데이터를 분석하며, PostgreSQL·MySQL·SQLite와 연동하는 법을 익힌다.

**6부 (17~19장)** 는 성능 최적화와 운영이다. 실행 계획을 읽는 법, 메모리와 병렬 처리를 튜닝하는 법, Docker 배포와 보안 설정을 다룬다.

**7부 (20~21장)** 는 실전 프로젝트다. S3 + Parquet 기반 데이터 레이크 구축, Python ML 파이프라인과의 통합 등 실제 업무에 가까운 시나리오를 직접 구현해본다.

각 장은 독립적으로 읽을 수 있도록 구성했다. 처음부터 순서대로 읽어도 좋고, 필요한 주제를 골라 읽어도 된다. 다만 DuckDB를 처음 접하는 독자라면 1부와 2부를 먼저 읽고 나머지를 탐색하길 권한다.

### 이 책에서 사용하는 환경

- DuckDB v1.x (Stable)
- Python 3.10 이상 (Python 예제 장에서 사용)
- 운영체제: macOS, Linux, Windows 모두 동작 확인

예제 코드와 데이터셋은 책의 GitHub 저장소에서 내려받을 수 있다.

> DuckDB가 그 도구가 되어줄 것이다.
