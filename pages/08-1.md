# 08-1. JSON 읽기·쓰기

### JSON 파일 직접 읽기

DuckDB는 JSON 파일을 마치 테이블처럼 FROM 절에 직접 지정할 수 있다. 별도의 로드 단계 없이 파일 경로만 지정하면 DuckDB가 구조를 자동으로 감지하고 쿼리를 실행한다.

```sql
-- 가장 간단한 방법: 파일을 직접 쿼리
SELECT * FROM 'data.json';
SELECT * FROM 'logs/events.json';

-- read_json_auto: 구조를 자동으로 감지
SELECT * FROM read_json_auto('data.json');

-- read_json: 옵션을 명시적으로 지정
SELECT * FROM read_json('data.json', format='auto');
```

### JSON 파일 형식

실무에서 JSON 파일은 단일 배열, 줄 단위 레코드(NDJSON), 또는 래퍼 객체 안에 배열이 들어간 형태 등 다양하게 존재한다. DuckDB는 세 가지 JSON 파일 형식을 지원하며, 형식을 명시하면 DuckDB가 올바른 파서를 선택해 처리한다.

```
┌──────────────────────────────────────────────────────────────┐
│                   JSON 파일 형식                              │
├──────────────────┬───────────────────────────────────────────┤
│ 형식              │ 예시                                       │
├──────────────────┼───────────────────────────────────────────┤
│ JSON Array       │ [{"id":1,"name":"Alice"},{"id":2,...}]    │
│ NDJSON/JSONL     │ {"id":1,"name":"Alice"}                   │
│ (한 줄에 하나)    │ {"id":2,"name":"Bob"}                     │
│ JSON Object      │ {"records": [...], "meta": {...}}         │
└──────────────────┴───────────────────────────────────────────┘
```

```sql
-- JSON Array (파일 전체가 배열)
SELECT * FROM read_json('data.json', format='array');

-- NDJSON / JSONL (줄마다 독립된 JSON 객체)
SELECT * FROM read_json('events.ndjson', format='newline_delimited');
SELECT * FROM read_json('logs.jsonl',    format='newline_delimited');

-- 자동 감지 (대부분의 경우 이것으로 충분)
SELECT * FROM read_json_auto('data.json');
```

> **팁:** 형식을 모를 때는 `read_json_auto()`를 먼저 시도한다. DuckDB가 파일 앞부분을 샘플링해 형식과 열 타입을 자동으로 추론하므로, 대부분의 경우 별도 옵션 없이도 올바르게 읽힌다.

### read_json 주요 옵션

스키마가 복잡하거나 타임스탬프 형식이 표준과 다를 때는 `read_json`에 옵션을 직접 지정한다. DuckDB는 여기서 지정한 열 타입과 파싱 규칙을 우선 적용하므로 자동 감지보다 정확하게 데이터를 가져올 수 있다.

```sql
SELECT * FROM read_json(
    'events.json',

    -- 형식
    format          = 'newline_delimited',  -- 'array', 'auto'

    -- 스키마
    columns         = {
        'id':         'INTEGER',
        'event_type': 'VARCHAR',
        'ts':         'TIMESTAMP',
        'payload':    'JSON'         -- 중첩 구조는 JSON 타입으로 유지
    },

    -- 타입 감지 샘플 크기
    sample_size     = 20480,        -- 타입 감지에 사용할 행 수

    -- 최대 JSON 깊이 (자동 감지 시 성능 튜닝)
    maximum_depth   = 4,

    -- 오류 처리
    ignore_errors   = true,

    -- 날짜 형식
    timestampformat = '%Y-%m-%dT%H:%M:%SZ',

    -- 파일 출처 추적
    filename        = true
);
```

### 여러 JSON 파일 한 번에 읽기

로그나 이벤트 데이터는 날짜별로 파일이 분리되어 있는 경우가 많다. DuckDB는 와일드카드와 파일 배열을 지원해, 여러 파일을 하나의 쿼리로 통합 조회할 수 있다. `filename=true` 옵션을 사용하면 각 행이 어느 파일에서 왔는지도 추적된다.

```sql
-- 와일드카드
SELECT * FROM 'logs/2024_*.json';
SELECT * FROM read_json_auto('events/*.jsonl');

-- 배열로 지정
SELECT * FROM read_json(['jan.json', 'feb.json', 'mar.json']);

-- 파일 출처 추적
SELECT filename, *
FROM read_json('logs/*.ndjson', filename=true);
-- filename            | id | event_type | ts
-- ────────────────────┼────┼────────────┼──────────────────
-- logs/jan.ndjson     |  1 | login      | 2024-01-05 ...
-- logs/feb.ndjson     | 42 | purchase   | 2024-02-11 ...
```

### JSON을 테이블로 적재

반복 쿼리가 많을 때는 JSON 파일을 DuckDB 테이블로 한 번만 적재해 두면 이후 쿼리 성능이 크게 향상된다. CTAS(CREATE TABLE AS SELECT)로 한 번에 생성하거나, 스키마를 먼저 정의한 뒤 `COPY`로 안전하게 불러올 수 있다.

```sql
-- CTAS: JSON에서 직접 테이블 생성
CREATE TABLE events AS
SELECT * FROM read_json_auto('events.ndjson');

-- 스키마를 명시해서 정확하게 적재
CREATE TABLE logs (
    id         BIGINT,
    ts         TIMESTAMP,
    level      VARCHAR,
    message    VARCHAR,
    context    JSON
);

COPY logs FROM 'app.ndjson' (FORMAT JSON);

-- INSERT INTO
INSERT INTO events
SELECT * FROM read_json_auto('new_events.json');
```

> **팁:** `COPY logs FROM ... (FORMAT JSON)`은 미리 정의된 스키마에 맞게 데이터를 강제 변환한다. 타입 불일치가 있으면 오류가 발생하므로, 불확실한 원본에는 먼저 `read_json_auto`로 구조를 확인한 뒤 스키마를 정의하는 순서를 권장한다.

### JSON 쓰기

분석 결과나 정제된 데이터를 JSON으로 내보낼 때는 `COPY ... TO` 구문을 사용한다. `ARRAY true/false` 옵션으로 JSON 배열 형식과 NDJSON 형식을 선택할 수 있고, GZIP 압축도 함께 지정할 수 있다.

```sql
-- 테이블 → JSON 파일
COPY employees TO 'employees.json' (FORMAT JSON);

-- 쿼리 결과 → JSON
COPY (
    SELECT id, name, department, salary
    FROM employees
    WHERE active = true
    ORDER BY department, name
) TO 'active_employees.json';

-- NDJSON 형식으로 출력
COPY employees TO 'employees.ndjson' (FORMAT JSON, ARRAY false);

-- JSON Array 형식으로 출력
COPY employees TO 'employees_array.json' (FORMAT JSON, ARRAY true);

-- 압축 출력
COPY employees TO 'employees.json.gz' (FORMAT JSON, COMPRESSION GZIP);

-- Python에서 처리 후 JSON 저장
```

```python
import duckdb, json

con = duckdb.connect()

# 쿼리 결과를 JSON으로
result = con.execute("""
    SELECT department, COUNT(*) AS cnt, AVG(salary) AS avg_sal
    FROM employees
    GROUP BY department
""").fetchdf()

result.to_json('dept_stats.json', orient='records', indent=2)

# DuckDB 직접 내보내기
con.execute("""
    COPY (SELECT * FROM employees)
    TO 'out.json' (FORMAT JSON, ARRAY true)
""")
```

JSON 파일을 읽고 쓰는 기초를 갖추었다면, 이제 JSON 데이터 안에서 원하는 값을 꺼내고 변환하는 방법으로 넘어간다.
