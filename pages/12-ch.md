# 제12장: Python과 DuckDB

Python은 DuckDB의 가장 중요한 클라이언트 환경이다. `pip install duckdb` 한 줄로 설치되고, 별도 서버 없이 프로세스 내에서 실행된다. Pandas·Polars·PyArrow와의 긴밀한 연동, 제로 카피(Zero-copy) 데이터 전달, Jupyter Notebook에서의 인터랙티브 분석까지 — Python 생태계와 DuckDB가 만나는 접점을 이 장에서 정리한다.

이 장은 크게 네 흐름으로 구성된다. 먼저 DuckDB를 Python 환경에 설치하고 연결하는 기본기를 다진 뒤, Pandas·Polars·PyArrow와의 데이터 교환 방식을 살펴본다. 그다음 SQL 문자열 없이 Python 메서드만으로 쿼리를 조합하는 관계형 API를 익히고, 마지막으로 Jupyter Notebook에서의 시각화와 인터랙티브 분석 패턴을 소개한다.

```python
import duckdb
import pandas as pd

con = duckdb.connect()

# Pandas DataFrame을 SQL에서 직접 참조 (Zero-copy)
df = pd.read_csv('sales.csv')

result = con.execute("""
    SELECT region, SUM(amount) AS total
    FROM df
    WHERE order_date >= '2024-01-01'
    GROUP BY region
    ORDER BY total DESC
""").df()
```

## 주요 학습 내용

- DuckDB는 Pandas/Polars DataFrame을 직접 SQL에서 테이블처럼 참조할 수 있다. 복사나 직렬화 없이 메모리를 공유하는 Zero-Copy 방식이라 대용량 데이터도 효율적으로 처리한다.
- `.fetch_arrow_table()`은 결과를 Arrow 포맷으로 반환한다. Arrow는 NumPy와 메모리 레이아웃이 호환되어 PyTorch·scikit-learn 같은 ML 프레임워크에 복사 없이 전달할 수 있다.
- `fetchmany(n)` 또는 `fetch_arrow_reader()`는 결과 전체를 메모리에 올리지 않고 스트리밍으로 처리한다. 수억 행을 다루는 파이프라인에서 메모리 오버플로를 예방하는 핵심 패턴이다.
