# 01-3. DuckDB UI 소개

DuckDB v1.1부터 **UI 확장(ui extension)** 이 공식 core extension으로 포함되었다. 이 확장은 로컬 DuckDB 인스턴스에 웹 기반 그래픽 인터페이스를 제공한다. SQL 편집기, 결과 테이블, 스키마 탐색기를 브라우저에서 바로 쓸 수 있어 CLI가 불편한 경우 유용한 대안이 된다.

> UI 확장은 MotherDuck에서 제작·관리하며, 별도 설치 없이 DuckDB에 기본 포함되어 있다.

---

### UI 실행하기

**방법 1: CLI 플래그**

```bash
duckdb -ui
```

**방법 2: SQL 명령**

```sql
CALL start_ui();
```

두 방법 모두 기본 브라우저에서 다음 주소를 자동으로 연다.

```
http://localhost:4213
```

브라우저를 자동으로 열지 않고 서버만 시작하려면 다음 명령을 쓴다.

```sql
CALL start_ui_server();
```

그 다음 브라우저에서 `http://localhost:4213` 을 직접 입력한다.

---

### 주요 기능

**로컬 우선(Local-first) 실행**
UI는 사용자의 로컬 머신에서 실행된다. "쿼리와 데이터는 명시적으로 MotherDuck에 연결하지 않는 한 컴퓨터를 벗어나지 않는다." 네이티브 프로세스로 동작하기 때문에 WebAssembly 기반 브라우저 내 DuckDB보다 빠르고, 모든 코어와 메모리를 활용할 수 있다.

**자동 컨텍스트 연결**
UI는 실행을 시작한 DuckDB 인스턴스에 자동으로 연결된다. CLI에서 이미 로드해둔 데이터나 테이블을 그대로 UI에서 쿼리할 수 있다.

---

### 포트 설정 변경

기본 포트 4213이 다른 프로세스와 충돌하는 경우 변경할 수 있다.

```sql
SET ui_local_port = 4214;
CALL start_ui();
```

| 설정 | 기본값 | 설명 |
|---|---|---|
| `ui_local_port` | 4213 | HTTP 서버 포트 |
| `ui_remote_url` | https://ui.duckdb.org | UI 파일 소스 |
| `ui_polling_interval` | 284ms | 백그라운드 업데이트 주기 |

---

### 제한 사항

- ARM 기반 Windows(`windows_arm64`)에서는 현재 지원되지 않는다.
- Read-only 모드로 직접 UI를 시작하는 것은 지원되지 않는다. 읽기 전용 데이터베이스는 UI 내부에서 `ATTACH ... (READ_ONLY)` 로 연결해야 한다.
