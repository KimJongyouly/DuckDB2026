# 03-4. 서브쿼리와 CTE

### 서브쿼리

단일 쿼리만으로 해결하기 어려운 문제 — 예를 들어 "전체 평균보다 급여가 높은 직원 목록" — 는 쿼리 결과를 다른 쿼리의 입력으로 전달하는 서브쿼리로 해결한다. 서브쿼리는 위치에 따라 역할이 달라진다.

```sql
-- WHERE 절 서브쿼리: 스칼라 비교
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- WHERE 절 서브쿼리: IN
SELECT name
FROM employees
WHERE dept_id IN (
    SELECT id FROM departments WHERE location = 'Seoul'
);

-- FROM 절 서브쿼리 (인라인 뷰)
SELECT dept, avg_sal
FROM (
    SELECT department AS dept, AVG(salary) AS avg_sal
    FROM employees
    GROUP BY department
) AS dept_avg
WHERE avg_sal > 70000;

-- SELECT 절 서브쿼리: 스칼라 서브쿼리
SELECT
    name,
    salary,
    (SELECT AVG(salary) FROM employees) AS company_avg,
    salary - (SELECT AVG(salary) FROM employees) AS diff
FROM employees;
```

### 상관 서브쿼리 (Correlated Subquery)

```sql
-- 각 직원의 급여가 같은 부서 평균보다 높은지 확인
SELECT name, department, salary
FROM employees e1
WHERE salary > (
    SELECT AVG(salary)
    FROM employees e2
    WHERE e2.department = e1.department  -- 외부 쿼리 참조
);

-- EXISTS: 존재 여부 확인
SELECT name
FROM departments d
WHERE EXISTS (
    SELECT 1 FROM employees e
    WHERE e.dept_id = d.id
    AND e.hire_date > '2023-01-01'
);
```

### CTE (Common Table Expression)

CTE는 WITH 절을 사용해 쿼리에 이름을 붙인 임시 결과 집합이다. 복잡한 쿼리를 단계적으로 분해할 때 유용하다.

```sql
-- 기본 CTE
WITH dept_avg AS (
    SELECT department, AVG(salary) AS avg_sal
    FROM employees
    GROUP BY department
)
SELECT e.name, e.salary, d.avg_sal
FROM employees e
JOIN dept_avg d ON e.department = d.department
WHERE e.salary > d.avg_sal;

-- 다중 CTE
WITH
active_employees AS (
    SELECT * FROM employees WHERE active = true
),
dept_stats AS (
    SELECT department, COUNT(*) AS cnt, AVG(salary) AS avg_sal
    FROM active_employees
    GROUP BY department
),
high_performing AS (
    SELECT * FROM dept_stats
    WHERE avg_sal > 80000
)
SELECT *
FROM high_performing
ORDER BY avg_sal DESC;
```

### 재귀 CTE

재귀 CTE는 계층 구조나 그래프 탐색에 사용한다.

```sql
-- 조직도 계층 탐색
WITH RECURSIVE org_tree AS (
    -- 기저 조건: 최상위 관리자 (manager_id가 없는 직원)
    SELECT id, name, manager_id, 1 AS depth, name AS path
    FROM employees
    WHERE manager_id IS NULL

    UNION ALL

    -- 재귀 조건: 하위 직원 추가
    SELECT e.id, e.name, e.manager_id, t.depth + 1, t.path || ' > ' || e.name
    FROM employees e
    JOIN org_tree t ON e.manager_id = t.id
)
SELECT depth, path, name
FROM org_tree
ORDER BY path;
```

```
재귀 CTE 실행 흐름
───────────────────────────────
1. 기저 조건 실행 → 초기 결과 집합
2. 결과 집합을 재귀 조건에 입력
3. 새 행 생성 → 결과 집합에 추가
4. 새 행 없으면 종료
```
