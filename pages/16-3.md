# 16-3. SQLite 연동

### 확장 설치

```sql
INSTALL sqlite; LOAD sqlite;
```

### ATTACH로 SQLite 연결

```sql
-- SQLite 파일 연결
ATTACH 'mydb.sqlite' AS sqlite_db (TYPE SQLITE);
ATTACH 'data/app.db' AS app_db (TYPE SQLITE);

-- 읽기 전용
ATTACH 'production.sqlite' AS sqlite_prod (TYPE SQLITE, READ_ONLY);

-- 절대 경로
ATTACH '/home/user/data/mydb.sqlite' AS sqlite_db (TYPE SQLITE);

-- 연결 확인
SHOW ALL TABLES IN sqlite_db;
```

### 테이블 탐색과 쿼리

```sql
-- 테이블 구조
DESCRIBE sqlite_db.users;

-- 기본 쿼리
SELECT * FROM sqlite_db.users LIMIT 10;

SELECT
    strftime('%Y-%m', created_at) AS month,
    COUNT(*) AS new_users
FROM sqlite_db.users
GROUP BY month
ORDER BY month;

-- SQLite + DuckDB 로컬 테이블 조인
SELECT
    u.username,
    u.email,
    COUNT(l.id) AS log_count,
    MAX(l.created_at) AS last_login
FROM sqlite_db.users u
LEFT JOIN sqlite_db.login_logs l ON u.id = l.user_id
GROUP BY u.id, u.username, u.email
ORDER BY log_count DESC
LIMIT 20;
```

### SQLite → DuckDB 마이그레이션

SQLite는 타입 시스템이 느슨(dynamic typing)해 마이그레이션 시 주의가 필요하다.

```sql
-- 구조 확인 (SQLite의 선언 타입 vs 실제 저장값)
SELECT typeof(id), typeof(name), typeof(created_at)
FROM sqlite_db.users LIMIT 1;

-- 안전한 마이그레이션: all_varchar 읽기 후 수동 변환
CREATE TABLE users_local AS
SELECT
    CAST(id AS INTEGER)         AS id,
    CAST(name AS VARCHAR)       AS name,
    CAST(email AS VARCHAR)      AS email,
    TRY_CAST(age AS INTEGER)    AS age,
    TRY_CAST(created_at AS TIMESTAMP) AS created_at,
    CASE WHEN is_active IN ('1','true','True') THEN true
         WHEN is_active IN ('0','false','False') THEN false
         ELSE NULL
    END                         AS is_active
FROM sqlite_db.users;

-- 데이터 검증
SELECT
    COUNT(*) AS total,
    COUNT(id) AS valid_id,
    COUNT(created_at) AS valid_ts,
    COUNT(*) - COUNT(id) AS null_ids
FROM users_local;
```

### SQLite 앱 데이터 분석

SQLite는 모바일 앱, 브라우저, 데스크톱 소프트웨어에 내장된 경우가 많다. DuckDB로 이 데이터를 꺼내 분석하는 패턴이 실용적이다.

```sql
-- 크롬 브라우저 히스토리 분석 (예시)
-- 파일 위치: ~/.config/google-chrome/Default/History
ATTACH '/home/user/.config/google-chrome/Default/History'
AS chrome (TYPE SQLITE, READ_ONLY);

-- 방문 도메인 통계
SELECT
    REGEXP_EXTRACT(url, 'https?://([^/]+)', 1) AS domain,
    COUNT(*) AS visit_count
FROM chrome.visits v
JOIN chrome.urls u ON v.url = u.id
GROUP BY domain
ORDER BY visit_count DESC
LIMIT 20;

-- iOS 건강 앱 데이터 분석 (백업에서 추출한 경우)
ATTACH 'healthdb.sqlite' AS health (TYPE SQLITE, READ_ONLY);

SELECT
    type,
    DATE_TRUNC('week', startDate::TIMESTAMP) AS week,
    AVG(value::DOUBLE) AS avg_value,
    MIN(value::DOUBLE) AS min_value,
    MAX(value::DOUBLE) AS max_value
FROM health.samples
WHERE type LIKE '%HeartRate%'
GROUP BY type, week
ORDER BY week;
```

### DuckDB → SQLite 쓰기

```sql
-- DuckDB 분석 결과를 SQLite로 내보내기
INSERT INTO sqlite_db.analytics_summary
SELECT date, region, revenue FROM local_summary;

-- 테이블 생성 후 삽입
CREATE TABLE sqlite_db.report AS
SELECT * FROM monthly_report;

-- SQLite 파일로 COPY
COPY (SELECT * FROM summary) TO 'report.sqlite';
-- (단, DuckDB→SQLite COPY는 제한적, INSERT가 더 안정적)
```

### SQLite 타입 매핑

```
SQLite 선언 타입 → DuckDB 타입 매핑
─────────────────────────────────────────────────────
SQLite             DuckDB
──────────────────  ────────────────────
INTEGER / INT      BIGINT
REAL               DOUBLE
TEXT               VARCHAR
BLOB               BLOB
NUMERIC            DECIMAL (근사)
없음 (NULL)         NULL

SQLite의 동적 타입 주의사항:
- 동일 열에 INTEGER와 TEXT가 섞일 수 있음
- TRY_CAST + COALESCE로 안전하게 처리
- 날짜/시간은 TEXT나 INTEGER(Unix epoch)로 저장
```
