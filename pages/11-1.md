# 11-1. Apache Iceberg

Apache Iceberg는 넷플릭스가 페타바이트 규모의 데이터를 안정적으로 관리하기 위해 설계한 오픈 테이블 포맷으로, 현재 Apache 최상위 프로젝트로 운영된다. 단순한 파일 포맷이 아니라, 파일들을 테이블처럼 다룰 수 있도록 하는 메타데이터 레이어다. DuckDB의 iceberg 확장을 설치하면 이 메타데이터 구조를 해석해 Iceberg 테이블을 SQL로 직접 쿼리할 수 있다.

### Iceberg 개요

```
┌────────────────────────────────────────────────────────────┐
│                  Apache Iceberg 레이어 구조                  │
├────────────────────────────────────────────────────────────┤
│  Catalog Layer                                             │
│  → 테이블의 현재 메타데이터 파일 경로를 가리키는 포인터       │
│  → REST Catalog, Hive Metastore, AWS Glue 등               │
├────────────────────────────────────────────────────────────┤
│  Metadata Layer                                            │
│  → metadata/*.json: 스냅샷 목록, 스키마 히스토리, 파티션 스펙│
│  → snapshots/: 각 커밋 시점의 상태                         │
├────────────────────────────────────────────────────────────┤
│  Manifest Layer                                            │
│  → manifest_list: 스냅샷에 속한 매니페스트 목록             │
│  → manifest files: 각 데이터 파일의 경로·통계·파티션 정보   │
├────────────────────────────────────────────────────────────┤
│  Data Layer                                                │
│  → Parquet / ORC / Avro 데이터 파일                        │
└────────────────────────────────────────────────────────────┘
```

### Iceberg 확장 설치

Iceberg 기능을 사용하려면 먼저 확장을 설치해야 한다. 한 번 설치하면 이후에는 `LOAD`만으로 활성화된다.

```sql
-- iceberg 확장 설치 및 로드
INSTALL iceberg;
LOAD iceberg;

-- 버전 확인
SELECT * FROM duckdb_extensions() WHERE extension_name = 'iceberg';
```

> **팁:** `INSTALL`은 최초 1회만 실행하면 된다. 이후 세션에서는 `LOAD iceberg;`만 실행하면 DuckDB가 로컬에 캐시된 확장을 로드한다.

### Iceberg 테이블 읽기

`iceberg_scan()`은 Iceberg 테이블의 메타데이터를 읽어 현재 스냅샷 기준의 데이터 파일 목록을 결정하고, 해당 Parquet 파일들을 병렬로 읽어 결과를 반환한다. 로컬 경로, S3, REST Catalog 모두 동일한 함수로 접근할 수 있다.

```sql
-- 로컬 Iceberg 테이블 읽기
SELECT * FROM iceberg_scan('path/to/iceberg_table');

-- 옵션 지정
SELECT * FROM iceberg_scan(
    'path/to/iceberg_table',
    allow_moved_paths = true    -- 경로가 바뀐 경우 허용
);

-- S3의 Iceberg 테이블 읽기
SELECT * FROM iceberg_scan('s3://my-bucket/warehouse/sales');

-- ATTACH로 카탈로그 연결
ATTACH 's3://my-bucket/warehouse' AS iceberg_catalog (TYPE ICEBERG);
SELECT * FROM iceberg_catalog.sales;
```

### Iceberg 메타데이터 조회

```sql
-- 스냅샷 목록 조회 (타임 트래블 기반)
SELECT * FROM iceberg_snapshots('path/to/iceberg_table');
-- snapshot_id        | timestamp_ms         | operation | ...
-- ───────────────────┼──────────────────────┼───────────┼───
-- 8312567490123456   | 2024-01-05 09:00:00  | append    |
-- 8312567490234567   | 2024-01-05 18:00:00  | overwrite |
-- 8312567490345678   | 2024-01-06 09:00:00  | append    |

-- 매니페스트 파일 목록
SELECT * FROM iceberg_manifests('path/to/iceberg_table');

-- 데이터 파일 목록
SELECT * FROM iceberg_files('path/to/iceberg_table');
-- content  | file_path              | file_format | record_count | ...
-- ─────────┼────────────────────────┼─────────────┼──────────────┼───
-- 0 (DATA) | data/00000-0-abc.parquet| PARQUET    | 50000        |
-- 0 (DATA) | data/00001-1-def.parquet| PARQUET    | 48300        |

-- 메타데이터 파일 직접 읽기 (고급)
SELECT * FROM iceberg_table_changes(
    'path/to/iceberg_table',
    snapshot_id1 = 8312567490123456,
    snapshot_id2 = 8312567490345678
);
```

### 타임 트래블

Iceberg의 스냅샷 기반 구조는 과거 데이터를 조회하는 타임 트래블을 자연스럽게 지원한다. 각 커밋은 독립된 스냅샷으로 보존되므로, 스냅샷 ID나 타임스탬프를 지정하면 DuckDB가 해당 시점의 메타데이터를 참조해 데이터를 반환한다.

```sql
-- 특정 스냅샷 ID로 조회
SELECT * FROM iceberg_scan(
    'path/to/iceberg_table',
    snapshot_id = 8312567490123456
);

-- 특정 시점으로 조회 (AS OF)
SELECT * FROM iceberg_scan(
    'path/to/iceberg_table',
    version = '2024-01-05 09:00:00'
);

-- 어제 데이터와 오늘 데이터 비교
WITH
today AS (
    SELECT * FROM iceberg_scan('path/to/orders')
),
yesterday AS (
    SELECT * FROM iceberg_scan(
        'path/to/orders',
        version = (CURRENT_TIMESTAMP - INTERVAL '1 day')::VARCHAR
    )
)
SELECT
    'today'     AS snap,
    COUNT(*)    AS rows,
    SUM(amount) AS total
FROM today
UNION ALL
SELECT 'yesterday', COUNT(*), SUM(amount)
FROM yesterday;
```

### REST Catalog 연결

```sql
-- REST Catalog (Polaris, Nessie, Unity Catalog 등)
ATTACH '' AS cat (
    TYPE         ICEBERG,
    ENDPOINT     'https://catalog.example.com/api/catalog',
    CREDENTIAL   'Bearer my_token',
    WAREHOUSE    'my_warehouse'
);

-- 카탈로그 탐색
SHOW ALL TABLES IN cat;

-- 테이블 쿼리
SELECT * FROM cat.my_namespace.my_table;
SELECT * FROM cat.my_namespace.my_table
WHERE order_date >= '2024-01-01';

-- 카탈로그를 통한 Iceberg 쓰기
INSERT INTO cat.my_namespace.sales
SELECT * FROM staging_sales;

CREATE TABLE cat.my_namespace.new_table AS
SELECT * FROM source_data;
```

### Iceberg 테이블 생성과 쓰기

```sql
-- 카탈로그 없이 로컬에 Iceberg 테이블 생성
-- (PyIceberg 또는 Spark로 초기 생성 후 DuckDB로 읽는 패턴이 일반적)

-- REST Catalog를 통한 생성
CREATE TABLE cat.analytics.daily_sales (
    id          BIGINT,
    order_date  DATE,
    region      VARCHAR,
    amount      DECIMAL(12, 2),
    customer_id INTEGER
) WITH (
    partitioning = ARRAY['month(order_date)', 'region']
);

-- 데이터 삽입
INSERT INTO cat.analytics.daily_sales
SELECT * FROM read_parquet('staging/*.parquet');

-- UPSERT (MERGE INTO)
MERGE INTO cat.analytics.daily_sales AS target
USING staging_updates AS src
ON target.id = src.id
WHEN MATCHED THEN
    UPDATE SET amount = src.amount, region = src.region
WHEN NOT MATCHED THEN
    INSERT VALUES (src.id, src.order_date, src.region, src.amount, src.customer_id);
```

### Python + Iceberg 패턴

```python
import duckdb

con = duckdb.connect()
con.execute("INSTALL iceberg; LOAD iceberg;")

# 로컬 Iceberg 테이블 읽기
df = con.execute("""
    SELECT region, SUM(amount) AS total
    FROM iceberg_scan('warehouse/sales')
    WHERE YEAR(order_date) = 2024
    GROUP BY region
    ORDER BY total DESC
""").df()

# PyIceberg와 연동 (테이블 생성은 PyIceberg, 쿼리는 DuckDB)
from pyiceberg.catalog import load_catalog

catalog = load_catalog('rest', **{
    'uri': 'https://catalog.example.com',
    'credential': 'my_token',
})
table = catalog.load_table('my_ns.sales')

# PyIceberg Table → DuckDB
con.register('iceberg_sales', table.scan().to_arrow())
result = con.execute("SELECT * FROM iceberg_sales LIMIT 100").df()

# DuckDB로 Iceberg 결과 쓰기 (PyIceberg 경유)
arrow_result = con.execute("SELECT * FROM iceberg_scan('warehouse/sales')").arrow()
catalog.load_table('my_ns.sales_copy').append(arrow_result)
```
