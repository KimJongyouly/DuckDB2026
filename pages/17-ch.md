# 제17장: 설정과 튜닝

DuckDB는 기본 설정만으로도 대부분의 워크로드를 충분히 처리하지만, 환경에 맞게 설정을 조정하면 훨씬 더 나은 성능과 안정성을 얻을 수 있다. 이 장에서는 DuckDB의 전역 설정 시스템, Pragma 명령, 그리고 자격증명을 안전하게 관리하는 Secrets Manager를 체계적으로 정리한다.

```sql
-- 가장 중요한 두 가지 설정
SET threads = 8;
SET memory_limit = '16GB';

-- 메모리 초과 시 스필 경로 (SSD 권장)
SET temp_directory = '/tmp/duckdb_spill';
```

## 주요 학습 내용

- `SET threads = N`은 DuckDB가 병렬 처리에 사용할 스레드 수를 결정한다. 기본값은 논리 CPU 수지만, 다른 프로세스와 자원을 공유하는 환경에서는 절반 정도로 줄이는 것이 안전하다.
- `SET memory_limit = '8GB'`를 설정하면 DuckDB가 이 한도를 초과하면 `temp_directory`로 디스크 스필을 시작한다. 메모리 초과 오류를 방지하는 첫 번째 방어선이다.
- `CREATE SECRET`으로 S3 자격증명을 객체화하면 연결 시마다 환경 변수를 확인하는 번거로움 없이 안전하게 클라우드 파일에 접근할 수 있다. `PERSISTENT` 옵션으로 세션이 끊겨도 유지된다.
