# 제21장: DuckDB로 ML 전처리

머신러닝 프로젝트에서 전처리는 전체 시간의 80%를 차지한다고 알려져 있다. 수억 행의 데이터를 Pandas로 불러와 피처를 만들다 보면 메모리 부족과 느린 속도에 금세 막히게 된다. 실제로 1억 행 규모의 집계 연산에서 Pandas는 수십 분이 걸리고 수십 GB의 메모리를 소비하는 반면, DuckDB는 동일 작업을 멀티스레드로 병렬 처리해 수 분 내에 완료하며 메모리 사용량도 훨씬 적다. DuckDB는 대용량 피처 엔지니어링을 SQL로 표현하고, 결과를 Arrow나 NumPy 형식으로 ML 프레임워크에 직접 전달해 이 병목을 제거한다.

## 주요 학습 내용

- DuckDB는 수억 행 규모의 집계 피처를 멀티스레드 SQL로 처리해 Pandas 대비 메모리와 속도 모두 유리하다
- 타겟 인코딩과 정규화 통계는 반드시 훈련 데이터에서만 계산하고 검증/테스트 세트에는 JOIN으로 적용해야 데이터 누수를 막을 수 있다
- `fetch_arrow_table()`을 활용한 Zero-Copy 로딩은 데이터 변환 오버헤드를 없애고 scikit-learn, XGBoost, PyTorch와의 연동을 단순화한다
