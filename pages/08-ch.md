# 제8장: JSON 파일

JSON은 API 응답, 로그 파일, NoSQL 내보내기에서 가장 자주 만나는 데이터 형식이다. DuckDB는 JSON을 단순한 문자열 or 파일로 취급하는 것이 아니라 데이터베이스 내부의 파일과 같은 정식데이터로 취급하고 다룬다. 파일을 직접 쿼리하고, 중첩 구조를 SQL로 탐색하며, 복잡한 배열을 행으로 펼치는 작업까지 별도 파서 없이 SQL 한 줄로 처리할 수 있다. 이 장에서는 JSON 파일을 읽고 쓰는 기본부터 시작해, 경로 연산자와 JSON 함수로 중첩 구조를 다루고, 실무 데이터를 관계형 테이블로 변환하는 패턴까지 순서대로 살펴본다.

```sql
-- 가장 간단한 방법: 파일을 직접 쿼리
SELECT * FROM 'data.json';

-- read_json_auto: 구조를 자동으로 감지
SELECT * FROM read_json_auto('data.json');

-- 필드 접근 연산자
-- -> : JSON 반환 (중간 탐색, 체이닝 가능)
-- ->> : VARCHAR 반환 (최종 값 추출)
SELECT
    data->>'name'            AS name_str,
    data->'address'->>'city' AS city
FROM events;
```

## 주요 학습 내용

- `->` 연산자는 JSON 타입을 반환하고, `->>`는 문자열(TEXT)을 반환한다. 값 비교나 집계에 사용할 때는 `->>`로 추출하거나 `CAST`를 명시해야 한다.
- 배열 인덱스는 0-based가 아닌 1-based다. `json[0]`이 아니라 `json[1]`이 첫 번째 원소다. 이 점이 다른 언어와 달라 실수하기 쉽다.
- 자주 쿼리하는 JSON 필드는 `GENERATED ALWAYS AS (json_extract_string(data, '$.field'))`로 가상 열을 만들어 인덱스 없이도 빠른 접근을 보장할 수 있다.
