# 11-2. Delta Lake

### Delta Lake 개요

```
┌────────────────────────────────────────────────────────────┐
│                  Delta Lake 구조                            │
├────────────────────────────────────────────────────────────┤
│  _delta_log/                                               │
│  ├── 00000000000000000000.json   (초기 커밋: 테이블 생성)   │
│  ├── 00000000000000000001.json   (INSERT 커밋)              │
│  ├── 00000000000000000002.json   (UPDATE 커밋)              │
│  ├── ...                                                   │
│  └── 00000000000000000010.checkpoint.parquet  (체크포인트) │
├────────────────────────────────────────────────────────────┤
│  part-00000-abc.snappy.parquet   (데이터 파일)              │
│  part-00001-def.snappy.parquet                             │
│  ...                                                       │
└────────────────────────────────────────────────────────────┘

트랜잭션 로그: JSON 파일에 각 커밋의 add/remove 액션 기록
체크포인트:    10번마다 Parquet로 로그 압축 (빠른 읽기)
```

### Delta Lake 확장 설치

```sql
INSTALL delta;
LOAD delta;

SELECT * FROM duckdb_extensions() WHERE extension_name = 'delta';
```

### Delta Lake 테이블 읽기

```sql
-- 기본 읽기
SELECT * FROM delta_scan('path/to/delta_table');

-- 로컬 파일
SELECT * FROM delta_scan('./delta_tables/sales');

-- S3
SELECT * FROM delta_scan('s3://my-bucket/delta/sales');

-- Azure Blob Storage
SELECT * FROM delta_scan('az://container/delta/sales');

-- 조건 포함
SELECT region, SUM(amount) AS total
FROM delta_scan('delta_tables/sales')
WHERE order_date >= '2024-01-01'
GROUP BY region;
```

### Delta Lake 메타데이터 조회

```sql
-- 테이블 히스토리 (커밋 로그)
SELECT * FROM delta_table_history('path/to/delta_table');
-- version | timestamp            | operation   | operationParameters
-- ────────┼──────────────────────┼─────────────┼───────────────────
-- 0       | 2024-01-01 00:00:00  | CREATE TABLE| {...}
-- 1       | 2024-01-05 09:00:00  | WRITE       | {mode: Append}
-- 2       | 2024-01-05 18:00:00  | WRITE       | {mode: Overwrite}
-- 3       | 2024-01-06 09:00:00  | MERGE       | {...}

-- 커밋 수 확인
SELECT COUNT(*) FROM delta_table_history('path/to/delta_table');

-- 파일 목록 (현재 버전)
SELECT * FROM delta_table_files('path/to/delta_table');
-- path                          | size    | partition_values
-- ──────────────────────────────┼─────────┼─────────────────
-- part-00000-abc.snappy.parquet | 1048576 | {year=2024}
```

### 타임 트래블

```sql
-- 버전 번호로 조회
SELECT * FROM delta_scan(
    'path/to/delta_table',
    version = 2
);

-- 타임스탬프로 조회 (해당 시점의 최신 버전)
SELECT * FROM delta_scan(
    'path/to/delta_table',
    timestamp = '2024-01-05 18:00:00'
);

-- 버전 간 변경 내역 비교
WITH
v1 AS (SELECT id, amount FROM delta_scan('path/to/orders', version=1)),
v2 AS (SELECT id, amount FROM delta_scan('path/to/orders', version=2))
SELECT
    COALESCE(v1.id, v2.id) AS id,
    v1.amount AS old_amount,
    v2.amount AS new_amount,
    v2.amount - v1.amount AS diff
FROM v1 FULL JOIN v2 ON v1.id = v2.id
WHERE v1.amount IS DISTINCT FROM v2.amount;
```

### Delta Lake 쓰기

DuckDB의 Delta 확장은 현재 읽기 중심이다. 쓰기는 `delta-rs` 라이브러리(Python/Rust)나 Spark와 연동하는 것이 일반적이다.

```python
# delta-rs로 Delta Lake 테이블 생성·쓰기
from deltalake import write_deltalake, DeltaTable
import pandas as pd
import duckdb

con = duckdb.connect()

# DuckDB 쿼리 결과 → Delta Lake로 저장
arrow_result = con.execute("""
    SELECT
        id,
        order_date,
        region,
        SUM(amount) AS daily_total
    FROM read_parquet('raw_sales/*.parquet')
    GROUP BY id, order_date, region
""").arrow()

write_deltalake(
    'delta_tables/sales_summary',
    arrow_result,
    mode='overwrite',               # 'append', 'overwrite', 'error'
    partition_by=['order_date'],
    schema_mode='overwrite'
)

# 다시 DuckDB로 읽기
df = con.execute("""
    SELECT * FROM delta_scan('delta_tables/sales_summary')
    WHERE order_date >= '2024-01-01'
""").df()

# 증분 추가
new_data = con.execute("SELECT * FROM new_batch").arrow()
write_deltalake(
    'delta_tables/sales_summary',
    new_data,
    mode='append'
)
```

```python
# UPSERT (delta-rs)
from deltalake import DeltaTable

dt = DeltaTable('delta_tables/sales_summary')

new_data = con.execute("SELECT * FROM updates").arrow()

(dt.merge(
    source=new_data,
    predicate='target.id = source.id',
    source_alias='source',
    target_alias='target'
)
.when_matched_update_all()
.when_not_matched_insert_all()
.execute())
```

### Iceberg vs Delta Lake 비교

```
┌────────────────────────────────────────────────────────────────┐
│              Apache Iceberg vs Delta Lake                       │
├──────────────────┬─────────────────────┬────────────────────── ┤
│ 항목              │ Iceberg             │ Delta Lake             │
├──────────────────┼─────────────────────┼────────────────────── ┤
│ 메타데이터 형식   │ JSON + Avro         │ JSON + Parquet 체크포인트│
│ 트랜잭션 로그     │ 스냅샷 기반         │ 커밋 로그 기반          │
│ 타임 트래블       │ 스냅샷 ID / 타임스탬프│ 버전 번호 / 타임스탬프  │
│ 카탈로그          │ REST, Hive, Glue    │ Unity Catalog, Hive    │
│ 주요 생태계        │ Spark, Flink, Trino │ Spark, Databricks      │
│ DuckDB 읽기       │ ✓ (iceberg ext)     │ ✓ (delta ext)          │
│ DuckDB 쓰기       │ ✓ (카탈로그 경유)   │ △ (delta-rs 권장)      │
│ 파티션 진화       │ ✓ (Hidden Partition)│ ✓                      │
│ 스키마 진화       │ ✓                   │ ✓                      │
│ Row-level Delete  │ ✓                   │ ✓                      │
└──────────────────┴─────────────────────┴────────────────────── ┘
```
