# 13-1. CLI 옵션과 설정

### 설치와 기본 실행

```bash
# 설치 (Linux/macOS)
curl -fsSL https://install.duckdb.org | sh

# 또는 직접 다운로드
wget https://github.com/duckdb/duckdb/releases/latest/download/duckdb_cli-linux-amd64.zip
unzip duckdb_cli-linux-amd64.zip
chmod +x duckdb
sudo mv duckdb /usr/local/bin/

# 버전 확인
duckdb --version
# v1.1.0 3c695d7ba9 2024-09-09 08:13:43 +0000
```

### 실행 모드

DuckDB CLI는 실행 목적에 따라 크게 세 가지 모드로 구분된다. 인터랙티브 탐색, SQL 직접 실행, 파일 기반 배치 실행 중 상황에 맞는 모드를 선택하면 불필요한 오버헤드 없이 원하는 작업을 수행할 수 있다.

```bash
# 인메모리 인터랙티브 셸 (데이터 저장 안 됨)
duckdb

# 파일 연결 인터랙티브 셸
duckdb mydb.duckdb
duckdb data/analytics.duckdb

# 읽기 전용
duckdb -readonly mydb.duckdb

# SQL 직접 실행 후 종료 (-c)
duckdb -c "SELECT 42 AS answer"
duckdb mydb.duckdb -c "SELECT COUNT(*) FROM employees"

# SQL 파일 실행 (-init 또는 <)
duckdb mydb.duckdb < query.sql
duckdb mydb.duckdb -init setup.sql

# 출력 없이 실행 (파이프라인용)
duckdb -c "COPY t TO 'out.parquet'" mydb.duckdb
```

> **팁:** `-readonly` 플래그를 사용하면 데이터베이스 파일에 대한 쓰기 잠금을 획득하지 않으므로, 다른 프로세스가 동시에 접근 중인 DB를 안전하게 조회할 때 유용하다.

### CLI 실행 플래그 전체

```bash
duckdb [옵션] [데이터베이스 파일]

옵션:
  -c SQL           SQL 문 실행 후 종료
  -f FILE          SQL 파일 실행 후 종료
  -init FILE       시작 시 SQL 파일 실행 (셸은 계속 실행)
  -readonly        읽기 전용 모드
  -unsigned        서명되지 않은 확장 로드 허용
  -no-stdin        표준 입력 비활성화
  -batch           배치 모드 (프롬프트·진행률 없음, 스크립트용)
  -json            출력을 JSON으로
  -csv             출력을 CSV로
  -column          출력을 컬럼 형식으로
  -line            출력을 라인 형식으로
  -list            출력을 리스트 형식으로
  -separator SEP   리스트 모드 구분자 지정
  -nullvalue STR   NULL 표현 문자열
  -newline SEP     행 구분자 (기본: \n)
  -bail            오류 발생 시 즉시 종료
  -echo            실행 전 SQL 에코 출력
  -header          헤더 행 출력 여부
  -noheader        헤더 없이 출력
  -stats           쿼리 통계 출력
  -s SQL           -c와 동일 (호환성)
  -version         버전 출력
  -help            도움말
```

### 출력 형식 제어

CLI의 출력 형식은 단순한 표시 방식 이상의 의미를 갖는다. 파이프라인에서 DuckDB 결과를 다른 도구(jq, awk, 파이썬 스크립트 등)로 전달할 때, 출력 형식이 곧 데이터 계약(interface)이 된다. 용도에 맞는 형식을 선택해두면 후속 처리 로직을 단순하게 유지할 수 있다.

```bash
# 기본 출력 (컬럼 정렬)
duckdb -c "SELECT * FROM 'sales.csv' LIMIT 3"
# ┌───────┬─────────────┬─────────┐
# │  id   │    date     │ amount  │
# │ int32 │    date     │ double  │
# ├───────┼─────────────┼─────────┤
# │     1 │ 2024-01-05  │ 15000.0 │
# └───────┴─────────────┴─────────┘

# CSV 출력
duckdb -csv -c "SELECT * FROM 'sales.csv' LIMIT 3"
# id,date,amount
# 1,2024-01-05,15000.0

# JSON 출력
duckdb -json -c "SELECT * FROM 'sales.csv' LIMIT 3"
# [{"id":1,"date":"2024-01-05","amount":15000.0},...]

# 헤더 없이 CSV
duckdb -csv -noheader -c "SELECT COUNT(*) FROM 'data.parquet'"
# 1500000

# NULL 표현 변경
duckdb -nullvalue "N/A" -csv -c "SELECT * FROM t WHERE id < 5"
```

> **팁:** 스크립트에서 쿼리 결과를 셸 변수로 받을 때는 `-csv -noheader` 조합이 가장 안정적이다. 기본 박스 형식은 테두리 문자가 포함되어 파싱이 어렵고, `-json`은 배열 파싱이 필요해 단순 값 추출에는 과하다.

### 인터랙티브 셸 도트 명령어

인터랙티브 셸에서는 SQL 외에도 셸 자체를 제어하는 메타 명령어가 제공된다. `.`으로 시작하는 이 도트(dot) 명령어들은 SQL 구문이 아니므로 세미콜론 없이 실행되며, 출력 모드 변경, 결과 파일 리다이렉트, SQL 파일 실행 등 반복 작업의 효율을 크게 높여 준다.

```sql
-- 도움말
.help

-- 출력 모드 변경
.mode column      -- 컬럼 정렬 (기본값)
.mode line        -- 키=값 형식 (열이 많을 때 유용)
.mode csv         -- CSV 형식
.mode json        -- JSON 형식
.mode table       -- 마크다운 테이블 형식
.mode tabs        -- 탭 구분
.mode box         -- 박스 그리기 문자
.mode duckbox     -- DuckDB 스타일 박스 (기본)
.mode latex       -- LaTeX 표 형식
.mode insert      -- INSERT SQL 형식으로 출력

-- 헤더 토글
.headers on
.headers off

-- NULL 표현
.nullvalue NULL
.nullvalue ""

-- 구분자 (list 모드에서)
.separator "|"
.separator "\t"

-- 출력을 파일로 리다이렉트
.output result.csv
SELECT * FROM sales;
.output stdout    -- 다시 화면으로

-- 파일 추가 (append)
.output result.csv append
SELECT * FROM new_sales;
.output stdout

-- SQL 파일 실행
.read setup.sql
.read queries/analysis.sql

-- 타이머 (쿼리 실행 시간)
.timer on
SELECT COUNT(*) FROM large_table;
-- Run Time: real 0.234 user 1.876 sys 0.043

-- 쿼리 취소
-- Ctrl+C

-- 셸 종료
.quit
.exit
-- 또는 Ctrl+D
```

> **팁:** `.output result.csv`로 리다이렉트한 뒤 작업이 끝나면 반드시 `.output stdout`으로 복원해야 한다. 복원하지 않으면 이후 모든 출력이 파일에 계속 쌓인다.

### 데이터베이스 탐색 명령어

```sql
-- 테이블 목록
.tables
.tables sales%    -- 패턴 매칭

-- 뷰 목록
SHOW VIEWS;

-- 테이블 구조
.schema employees
DESCRIBE employees;
PRAGMA table_info('employees');

-- 모든 객체
SHOW ALL TABLES;

-- 인덱스 목록
PRAGMA show_indexes('employees');

-- 데이터베이스 정보
PRAGMA database_list;
SELECT * FROM duckdb_tables();
SELECT * FROM duckdb_columns() WHERE table_name = 'employees';
SELECT * FROM duckdb_settings();

-- 확장 목록
SELECT * FROM duckdb_extensions();
```

### 설정 영구화 (.duckdbrc)

셸 시작 시 자동으로 실행할 명령어를 `~/.duckdbrc`에 저장한다.

```sql
-- ~/.duckdbrc
.mode duckbox
.timer on
.headers on
.nullvalue NULL

-- 자주 쓰는 설정
SET threads = 8;
SET memory_limit = '8GB';
SET enable_progress_bar = true;

-- 자주 쓰는 뷰나 매크로 (인메모리 세션용)
-- LOAD 확장
LOAD json;
LOAD parquet;
```

```bash
# 특정 rc 파일 지정
duckdb -init ~/.duckdbrc_work mydb.duckdb
```

### 확장 관리

```sql
-- 확장 설치
INSTALL json;
INSTALL parquet;
INSTALL httpfs;
INSTALL spatial;
INSTALL iceberg;
INSTALL delta;
INSTALL excel;

-- 로드
LOAD json;
LOAD parquet;

-- 자동 로드 설정
SET autoload_known_extensions = true;
SET autoinstall_known_extensions = true;

-- 오프라인 환경: 로컬 확장 로드
LOAD '/path/to/extension.duckdb_extension';

-- 확장 업데이트
UPDATE EXTENSIONS;
```
