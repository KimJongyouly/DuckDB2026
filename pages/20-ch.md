# 제20장: 데이터 분석 파이프라인 구축

DuckDB의 진정한 가치는 파이프라인 안에서 드러난다. 로컬 머신 한 대에서 수십 GB의 데이터를 처리하고, S3 위의 Parquet 레이크를 직접 쿼리하며, 결과를 실시간 대시보드로 연결하는 전체 흐름을 DuckDB 하나로 관통할 수 있다. 이 장에서는 실제 운영 가능한 세 가지 파이프라인 패턴을 단계적으로 구축한다.

로컬 레이크에서 구조를 검증하고, S3 레이크로 확장하며, 최종적으로 Gold 마트를 대시보드에 연결하는 흐름이다. 각 섹션은 이전 단계의 결과물을 입력으로 사용하므로 순서대로 읽는 것을 권장한다.

```
[원본 데이터]
    │  CSV / JSON / 외부 스트림
    ▼
┌─────────────────────────────┐
│  20.1  로컬 데이터 레이크   │  Bronze → Silver → Gold
│  (메달리온 아키텍처)         │  로컬 파일시스템 + Parquet
└──────────────┬──────────────┘
               │  파이프라인 검증 완료 후
               ▼
┌─────────────────────────────┐
│  20.2  S3 + Parquet 레이크  │  클라우드 확장
│  (서버리스 ETL)              │  httpfs + Hive 파티셔닝
└──────────────┬──────────────┘
               │  Gold 마트 준비 완료 후
               ▼
┌─────────────────────────────┐
│  20.3  대시보드 연동         │  Streamlit / FastAPI
│  (시각화·자동 갱신)          │  Grafana / Superset
└─────────────────────────────┘
```

## 주요 학습 내용

- Bronze는 원본을 절대 변형하지 않는다. 재처리 가능성이 파이프라인 안정성의 핵심이다.
- S3 비용 최적화는 쿼리 작성 방식에서 결정된다. Hive 파티셔닝과 `WHERE` 조건을 함께 써야 파일 스캔을 줄일 수 있다.
- 대시보드 연결은 Gold 마트를 통해 이루어지며, 연결 방식과 무관하게 캐시 전략이 응답 속도를 좌우한다.
