# 17-3. Secrets Manager

Secrets Manager는 DuckDB 0.10.0에서 도입된 자격증명 관리 시스템이다. S3, Azure, GCS 등 외부 서비스의 인증 정보를 중앙에서 관리하며, 연결 문자열에 직접 비밀번호를 노출하지 않아도 된다.

### Secret 기본 개념

```
┌──────────────────────────────────────────────────────────────┐
│                     Secrets Manager                          │
│                                                              │
│  ┌────────────────┐   ┌────────────────┐  ┌──────────────┐  │
│  │   S3 Secret    │   │  Azure Secret  │  │  PG Secret   │  │
│  │  (TYPE: S3)    │   │ (TYPE: AZURE)  │  │(TYPE: POSTGRES│  │
│  │  SCOPE: s3://  │   │ SCOPE: azure://│  │SCOPE: host)  │  │
│  └────────────────┘   └────────────────┘  └──────────────┘  │
│                                                              │
│  저장 위치: 세션(기본) | ~/.duckdb/secrets/ (PERSISTENT)     │
└──────────────────────────────────────────────────────────────┘
```

### Secret 생성 기본 문법

```sql
CREATE [OR REPLACE] [PERSISTENT] SECRET [IF NOT EXISTS] secret_name (
    TYPE secret_type,
    [provider,]
    key value,
    ...
    [SCOPE scope_prefix]
);
```

### S3 자격증명 관리

```sql
-- 명시적 키 방식
CREATE OR REPLACE SECRET s3_main (
    TYPE S3,
    KEY_ID 'AKIAIOSFODNN7EXAMPLE',
    SECRET 'wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY',
    REGION 'ap-northeast-2'
);

-- IAM 역할 (EC2/ECS 인스턴스 메타데이터)
CREATE OR REPLACE SECRET s3_iam (
    TYPE S3,
    PROVIDER CREDENTIAL_CHAIN
);

-- 환경 변수 체인 (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY)
CREATE OR REPLACE SECRET s3_env (
    TYPE S3,
    PROVIDER ENV
);

-- 여러 버킷에 대해 각각 다른 자격증명
CREATE OR REPLACE SECRET s3_prod (
    TYPE S3,
    KEY_ID 'PROD_KEY_ID',
    SECRET 'PROD_SECRET',
    REGION 'us-east-1',
    SCOPE 's3://prod-data-bucket'   -- 이 버킷에만 적용
);

CREATE OR REPLACE SECRET s3_dev (
    TYPE S3,
    KEY_ID 'DEV_KEY_ID',
    SECRET 'DEV_SECRET',
    REGION 'us-west-2',
    SCOPE 's3://dev-data-bucket'   -- 이 버킷에만 적용
);

-- 생성 후 사용 (투명하게 자격증명 주입)
SELECT * FROM read_parquet('s3://prod-data-bucket/sales/2024/*.parquet');
SELECT * FROM read_parquet('s3://dev-data-bucket/test/*.parquet');
```

### S3 호환 스토리지 (MinIO, Cloudflare R2, GCS)

```sql
-- MinIO (자체 호스팅 S3 호환)
CREATE OR REPLACE SECRET minio_secret (
    TYPE S3,
    KEY_ID 'minioadmin',
    SECRET 'minioadmin',
    ENDPOINT 'localhost:9000',
    URL_STYLE 'path',        -- MinIO는 path-style URL 사용
    USE_SSL false
);

-- Cloudflare R2
CREATE OR REPLACE SECRET r2_secret (
    TYPE S3,
    KEY_ID 'your-r2-access-key-id',
    SECRET 'your-r2-secret',
    ENDPOINT 'your-account-id.r2.cloudflarestorage.com',
    REGION 'auto'
);

-- Google Cloud Storage (HMAC 키 방식)
CREATE OR REPLACE SECRET gcs_secret (
    TYPE GCS,
    KEY_ID 'GOOGXXXXXXXXXXXXXXXX',
    SECRET 'your-hmac-secret'
);
```

### Azure 자격증명 관리

```sql
-- 연결 문자열 방식
CREATE OR REPLACE SECRET azure_conn (
    TYPE AZURE,
    CONNECTION_STRING 'DefaultEndpointsProtocol=https;AccountName=myaccount;AccountKey=...'
);

-- 계정 키 방식
CREATE OR REPLACE SECRET azure_key (
    TYPE AZURE,
    ACCOUNT_NAME 'myaccount',
    ACCOUNT_KEY 'base64encodedkey=='
);

-- SAS 토큰 방식
CREATE OR REPLACE SECRET azure_sas (
    TYPE AZURE,
    ACCOUNT_NAME 'myaccount',
    SAS_TOKEN '?sv=2021-06-08&ss=b&srt=sco&sp=rwdlacupiytfx&...',
    SCOPE 'azure://myaccount.blob.core.windows.net/container'
);

-- Managed Identity (Azure VM / App Service)
CREATE OR REPLACE SECRET azure_mi (
    TYPE AZURE,
    PROVIDER CREDENTIAL_CHAIN,  -- 환경에 따라 자동 탐색
    ACCOUNT_NAME 'myaccount'
);

-- 사용
SELECT * FROM read_parquet('azure://mycontainer/data/*.parquet');
SELECT * FROM read_parquet('abfss://container@account.dfs.core.windows.net/path/');
```

### PostgreSQL Secret

```sql
-- PostgreSQL 자격증명을 Secret으로 관리
CREATE OR REPLACE SECRET pg_prod_secret (
    TYPE POSTGRES,
    HOST 'prod-db.example.com',
    PORT 5432,
    DATABASE 'mydb',
    USER 'analyst',
    PASSWORD 'secret_password',
    SCOPE 'prod-db.example.com'
);

-- Secret 기반 ATTACH (비밀번호 노출 없이 연결)
ATTACH '' AS pg_prod (TYPE POSTGRES, SECRET pg_prod_secret);
```

### Secret 영속화 (PERSISTENT)

기본적으로 Secret은 세션이 종료되면 사라진다. `PERSISTENT` 옵션을 사용하면 `~/.duckdb/secrets/` 디렉터리에 저장되어 다음 세션에서도 재사용할 수 있다.

```sql
-- 영속 Secret 생성
CREATE PERSISTENT SECRET s3_permanent (
    TYPE S3,
    PROVIDER CREDENTIAL_CHAIN
);

-- 영속 Secret 목록 확인
FROM duckdb_secrets() SELECT name, type, is_persistent, scope;
-- ┌──────────────┬──────┬───────────────┬───────────────────────┐
-- │     name     │ type │ is_persistent │        scope          │
-- ├──────────────┼──────┼───────────────┼───────────────────────┤
-- │ s3_permanent │  S3  │     true      │ s3://                 │
-- │ s3_main      │  S3  │     false     │ s3://                 │
-- └──────────────┴──────┴───────────────┴───────────────────────┘

-- Secret 삭제
DROP SECRET s3_main;
DROP PERSISTENT SECRET s3_permanent;
```

### Python에서 Secret 관리

```python
import duckdb
import os

con = duckdb.connect()

# 환경 변수에서 자격증명을 읽어 Secret 생성
con.execute(f"""
    CREATE OR REPLACE SECRET s3_secret (
        TYPE S3,
        KEY_ID '{os.environ["AWS_ACCESS_KEY_ID"]}',
        SECRET '{os.environ["AWS_SECRET_ACCESS_KEY"]}',
        REGION '{os.environ.get("AWS_DEFAULT_REGION", "us-east-1")}'
    )
""")

# 또는 credential_chain으로 환경 변수 자동 탐색
con.execute("""
    CREATE OR REPLACE SECRET s3_auto (
        TYPE S3,
        PROVIDER CREDENTIAL_CHAIN
    )
""")

# 이후 S3 경로를 투명하게 사용
df = con.execute("""
    SELECT region, SUM(amount) AS total
    FROM read_parquet('s3://my-bucket/sales/2024/**/*.parquet')
    GROUP BY region
""").df()
```

### Secret 우선순위와 SCOPE 매칭

여러 Secret이 존재할 때, DuckDB는 가장 구체적인 SCOPE를 가진 Secret을 선택한다.

```sql
-- 범용 S3 자격증명
CREATE SECRET s3_default (
    TYPE S3,
    PROVIDER CREDENTIAL_CHAIN,
    SCOPE 's3://'
);

-- 특정 버킷용 자격증명 (더 구체적이므로 우선 적용)
CREATE SECRET s3_finance (
    TYPE S3,
    KEY_ID 'FINANCE_KEY',
    SECRET 'FINANCE_SECRET',
    REGION 'ap-northeast-2',
    SCOPE 's3://finance-data-bucket'
);

-- 접근 시 SCOPE 기반 자동 선택
-- 이 쿼리는 s3_finance Secret 사용
SELECT * FROM read_parquet('s3://finance-data-bucket/reports/*.parquet');

-- 이 쿼리는 s3_default Secret 사용
SELECT * FROM read_parquet('s3://other-bucket/data/*.parquet');
```

### 보안 모범 사례

```
┌──────────────────────────────────────────────────────────────────┐
│                    Secret 보안 체크리스트                          │
├──────────────────────────────────────────────────────────────────┤
│ ✓ 연결 문자열에 비밀번호 하드코딩 금지                             │
│ ✓ CREDENTIAL_CHAIN으로 IAM/환경 변수 우선 사용                    │
│ ✓ PERSISTENT Secret 파일 권한 확인 (~/.duckdb/secrets/)           │
│ ✓ SCOPE를 최소 범위로 제한 (전체 s3:// 대신 특정 버킷)            │
│ ✓ 환경별 Secret 분리 (prod/dev/staging 별도 키)                   │
│ ✓ Secret 로그 출력 차단 (duckdb_secrets()는 키를 마스킹함)        │
│ ✓ CI/CD에서는 환경 변수 → CREDENTIAL_CHAIN 방식 사용              │
│ ✓ 최소 권한 원칙: 분석용 키는 읽기 전용 IAM 정책 부여             │
└──────────────────────────────────────────────────────────────────┘
```

```python
# 안전한 패턴: 환경 변수 + CREDENTIAL_CHAIN
import duckdb

con = duckdb.connect()

# 개발 환경: 명시적 키 (환경 변수에서 읽기)
if os.environ.get('ENV') == 'development':
    con.execute("""
        CREATE OR REPLACE SECRET s3 (
            TYPE S3,
            KEY_ID $key_id,
            SECRET $secret,
            REGION $region
        )
    """, {
        'key_id': os.environ['AWS_ACCESS_KEY_ID'],
        'secret': os.environ['AWS_SECRET_ACCESS_KEY'],
        'region': os.environ.get('AWS_DEFAULT_REGION', 'us-east-1'),
    })
else:
    # 프로덕션: IAM 역할 자동 감지
    con.execute("""
        CREATE OR REPLACE SECRET s3 (
            TYPE S3,
            PROVIDER CREDENTIAL_CHAIN
        )
    """)

# 이후 S3 경로를 코드에서 직접 사용 (자격증명 노출 없음)
result = con.execute("SELECT COUNT(*) FROM read_parquet('s3://bucket/data/*.parquet')").fetchone()
print(f"총 레코드: {result[0]:,}")
```
