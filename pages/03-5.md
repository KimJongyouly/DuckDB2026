# 03-5. WINDOW 함수와 QUALIFY

### 윈도우 함수 개요

윈도우 함수는 현재 행과 연관된 행들의 집합(윈도우)에 대해 계산을 수행한다. 집계 함수와 달리 결과 행 수가 줄지 않는다.

```
일반 집계 함수         윈도우 함수
────────────────      ───────────────────
원본: 5행  →          원본: 5행
결과: 1행              결과: 5행 (각 행에 집계값 추가)
```

```sql
-- 기본 문법
함수명() OVER (
    [PARTITION BY 열]   -- 그룹 분할
    [ORDER BY 열]       -- 정렬 기준
    [frame_clause]      -- 행 범위 지정
)
```

### 순위 함수

```sql
SELECT
    name,
    department,
    salary,
    -- 동점 있으면 같은 순위, 다음 순위 건너뜀 (1,1,3,4)
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank,
    -- 동점 있어도 순위 건너뛰지 않음 (1,1,2,3)
    DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dense_rank,
    -- 항상 고유한 순위 (1,2,3,4)
    ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS row_num,
    -- 백분위 순위 (0~1)
    PERCENT_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS pct_rank,
    -- 동일 비율 그룹 (N개 버킷으로 분할)
    NTILE(4) OVER (ORDER BY salary DESC) AS quartile
FROM employees;
```

### 집계 윈도우 함수

```sql
SELECT
    name,
    department,
    salary,
    -- 부서 내 급여 합계
    SUM(salary) OVER (PARTITION BY department) AS dept_total,
    -- 부서 내 평균 급여
    AVG(salary) OVER (PARTITION BY department) AS dept_avg,
    -- 누적 합계 (ORDER BY 포함)
    SUM(salary) OVER (
        PARTITION BY department
        ORDER BY hire_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS running_total,
    -- 이동 평균 (최근 3행)
    AVG(salary) OVER (
        ORDER BY hire_date
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS moving_avg_3
FROM employees;
```

### 이동/탐색 함수

```sql
SELECT
    date,
    revenue,
    -- 이전 행 값
    LAG(revenue, 1) OVER (ORDER BY date) AS prev_revenue,
    -- 다음 행 값
    LEAD(revenue, 1) OVER (ORDER BY date) AS next_revenue,
    -- 전일 대비 증감률
    (revenue - LAG(revenue) OVER (ORDER BY date))
        / LAG(revenue) OVER (ORDER BY date) * 100 AS growth_pct,
    -- 파티션의 첫 번째 값
    FIRST_VALUE(revenue) OVER (
        PARTITION BY YEAR(date) ORDER BY date
    ) AS year_first,
    -- 파티션의 마지막 값
    LAST_VALUE(revenue) OVER (
        PARTITION BY YEAR(date) ORDER BY date
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS year_last
FROM daily_sales;
```

### 프레임 절 (Frame Clause)

```sql
-- ROWS: 물리적 행 수 기준
ROWS BETWEEN 3 PRECEDING AND CURRENT ROW
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW  -- 누적
ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING          -- 전후 1행

-- RANGE: 값 범위 기준 (ORDER BY 열의 값)
RANGE BETWEEN INTERVAL '7 days' PRECEDING AND CURRENT ROW

-- GROUPS: 동일 ORDER BY 값 그룹 기준
GROUPS BETWEEN 1 PRECEDING AND CURRENT ROW
```

```
프레임 범위 시각화 (ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)
───────────────────────────────────────────────────────────
행 1:  [■]              (현재 행만)
행 2:  [■, ■]           (이전 1행 + 현재)
행 3:  [■, ■, ■]        (이전 2행 + 현재)  ← 여기서 고정
행 4:     [■, ■, ■]     (이전 2행 + 현재)
행 5:        [■, ■, ■]  (이전 2행 + 현재)
```

### QUALIFY 절 (DuckDB / BigQuery 전용)

QUALIFY는 윈도우 함수 결과로 행을 필터링한다. 기존에는 서브쿼리나 CTE를 써야 했던 패턴을 한 번에 처리한다.

```sql
-- 전통적인 방법: CTE 필요
WITH ranked AS (
    SELECT *,
           ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS rn
    FROM employees
)
SELECT * FROM ranked WHERE rn = 1;

-- QUALIFY 사용: 한 쿼리로 처리
SELECT *
FROM employees
QUALIFY ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) = 1;

-- 부서별 급여 상위 3명
SELECT name, department, salary
FROM employees
QUALIFY RANK() OVER (PARTITION BY department ORDER BY salary DESC) <= 3;
```
