# 15-2. httpfs: HTTP/S3 파일 직접 쿼리

`httpfs` 확장은 HTTP(S) URL과 S3 경로를 로컬 파일처럼 쿼리할 수 있게 한다. 파일을 다운로드하지 않고 HTTP Range 요청으로 필요한 청크만 읽는다.

### 기본 HTTP 쿼리

```sql
LOAD httpfs;

-- HTTP URL의 Parquet 파일 직접 쿼리
SELECT *
FROM read_parquet('https://example.com/data/sales.parquet')
LIMIT 100;

-- 메타데이터만 확인 (파일 전체 다운로드 없음)
DESCRIBE SELECT * FROM
    read_parquet('https://example.com/data/sales.parquet');

SELECT COUNT(*)
FROM read_parquet('https://example.com/data/sales.parquet');

-- CSV
SELECT *
FROM read_csv('https://example.com/data/employees.csv', header=true)
LIMIT 10;

-- JSON
SELECT *
FROM read_json('https://api.example.com/data/events.ndjson',
               format='newline_delimited')
LIMIT 5;

-- 공개 데이터셋 예시 (NYC Taxi 데이터)
SELECT
    passenger_count,
    AVG(trip_distance) AS avg_distance,
    AVG(total_amount) AS avg_fare
FROM read_parquet(
    'https://d37ci6vzurychx.cloudfront.net/trip-data/yellow_tripdata_2024-01.parquet'
)
GROUP BY passenger_count
ORDER BY passenger_count;
```

### HTTP Range 요청의 동작 원리

```
HTTP Range 요청으로 Parquet 읽기
────────────────────────────────────────────────────────────
1. DuckDB가 파일 끝 8바이트 요청 → Footer 크기 확인
   GET /sales.parquet  Range: bytes=-8

2. Footer 읽기 → Row Group 오프셋·열 통계 파악
   GET /sales.parquet  Range: bytes=<footer_offset>-<end>

3. WHERE 절로 필요한 Row Group만 선택 (Zone Map)
   GET /sales.parquet  Range: bytes=<rg2_start>-<rg2_end>

→ 1GB 파일에서 필요한 10MB만 다운로드
```

### S3 설정과 쿼리

```sql
LOAD httpfs;

-- S3 인증 설정
SET s3_region = 'ap-northeast-2';
SET s3_access_key_id = 'AKIAIOSFODNN7EXAMPLE';
SET s3_secret_access_key = 'wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY';

-- 퍼블릭 버킷 (인증 없음)
SELECT * FROM read_parquet('s3://my-public-bucket/data/sales.parquet');

-- 프라이빗 버킷
SELECT *
FROM read_parquet('s3://my-private-bucket/warehouse/sales/*.parquet')
LIMIT 10;

-- 와일드카드
SELECT COUNT(*)
FROM read_parquet('s3://bucket/data/year=2024/**/*.parquet');

-- Hive 파티셔닝
SELECT *
FROM read_parquet(
    's3://bucket/warehouse/**/*.parquet',
    hive_partitioning = true
)
WHERE year = 2024 AND month = 3;

-- S3 결과 저장
COPY (SELECT * FROM employees) TO 's3://bucket/output/employees.parquet'
(FORMAT PARQUET, COMPRESSION ZSTD);
```

### IAM 역할 기반 인증 (EC2/ECS)

```sql
-- EC2 인스턴스 프로파일 사용 (키 하드코딩 불필요)
SET s3_region = 'ap-northeast-2';
-- access key 설정 없이 인스턴스 메타데이터에서 자동으로 읽음

-- 환경 변수에서 자동으로 읽기
-- AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_DEFAULT_REGION
-- 이 환경 변수가 설정되어 있으면 별도 SET 불필요
```

### S3 호환 스토리지 (MinIO, Cloudflare R2 등)

```sql
-- MinIO 또는 기타 S3 호환 스토리지
SET s3_endpoint = 'minio.internal:9000';
SET s3_access_key_id = 'minioadmin';
SET s3_secret_access_key = 'minioadmin';
SET s3_use_ssl = false;
SET s3_url_style = 'path';          -- path 또는 vhost

SELECT * FROM read_parquet('s3://mybucket/data/sales.parquet');

-- Cloudflare R2
SET s3_endpoint = '<accountid>.r2.cloudflarestorage.com';
SET s3_access_key_id = 'r2_access_key';
SET s3_secret_access_key = 'r2_secret_key';
SET s3_region = 'auto';

-- Google Cloud Storage (GCS)
-- HMAC 키 사용
SET s3_endpoint = 'storage.googleapis.com';
SET s3_access_key_id = '<HMAC_access_key>';
SET s3_secret_access_key = '<HMAC_secret>';

SELECT * FROM read_parquet('s3://my-gcs-bucket/data/*.parquet');
```

### httpfs 성능 튜닝

```sql
-- HTTP 요청 병렬도 조정
SET http_retries = 3;
SET http_retry_wait_ms = 100;
SET http_timeout = 30000;       -- ms

-- S3 멀티파트 업로드 설정
SET s3_uploader_max_parts_per_file = 100;
SET s3_uploader_min_part_size = 5242880;     -- 5MB
SET s3_uploader_max_filesize = 10737418240;  -- 10GB

-- 글로벌 파일 캐시 (재쿼리 시 속도 향상)
SET enable_object_cache = true;
SET object_cache_size = '1GB';
```
