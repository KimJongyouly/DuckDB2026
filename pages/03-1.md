# 03-1. DuckDB SQL 개요

### 표준 SQL과의 관계

DuckDB는 SQL:2003 표준을 기반으로 하며, 분석 워크로드에 유용한 확장을 추가했다. 기본적인 CRUD 문법은 다른 데이터베이스와 거의 동일하지만, 몇 가지 DuckDB만의 특징이 있다. 이 특징들은 분석 쿼리를 더 짧고 읽기 쉽게 작성하도록 설계되었으며, 기존 SQL 경험을 그대로 활용하면서 점진적으로 익힐 수 있다.

```
┌─────────────────────────────────────────────────────┐
│                  DuckDB SQL 레이어                   │
├─────────────────────────────────────────────────────┤
│  표준 SQL (SELECT, INSERT, UPDATE, DELETE, ...)      │
├─────────────────────────────────────────────────────┤
│  PostgreSQL 호환 확장 (RETURNING, ILIKE, ...)        │
├─────────────────────────────────────────────────────┤
│  DuckDB 전용 문법 (QUALIFY, SAMPLE, PIVOT, ...)      │
└─────────────────────────────────────────────────────┘
```

### DuckDB SQL의 주요 특징

| 특징 | 설명 |
|------|------|
| 열 이름 대소문자 무감지 | `SELECT Name`과 `SELECT name`은 동일하게 처리 |
| 후행 쉼표 허용 | `SELECT a, b, FROM t` — 마지막 열 뒤 쉼표 허용 |
| FROM-first 문법 | `FROM t SELECT a, b` 순서로도 작성 가능 |
| 자동 타입 추론 | 리터럴 값에서 타입을 자동으로 추론 |
| 중첩 구조 지원 | LIST, STRUCT, MAP 등 복합 타입 기본 지원 |

### 쿼리 실행 방법

DuckDB는 두 가지 방식으로 쿼리를 실행할 수 있다. `.sql()`은 Relation 객체를 반환해 메서드 체이닝이 가능하고, `.execute()`는 커서 기반으로 동작해 기존 DB-API 2.0 방식과 호환된다.

```python
import duckdb

con = duckdb.connect()

# .sql() — Relation 객체 반환, 메서드 체이닝 가능
rel = con.sql("SELECT 42 AS answer")
rel.show()

# .execute() — 커서 기반, fetchall() / fetchdf() 등 사용
cur = con.execute("SELECT 42 AS answer")
print(cur.fetchall())
```
