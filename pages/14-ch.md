# 제14장: 기타 클라이언트

DuckDB는 Python과 CLI 외에도 Java, Node.js, Go, Rust, R, ODBC/ADBC, 그리고 브라우저의 WebAssembly까지 폭넓은 클라이언트를 공식 지원한다. 언어마다 API 스타일은 다르지만 핵심 개념—연결, 쿼리 실행, 파라미터화, 결과 변환—은 동일하다. 이 장에서는 각 클라이언트의 설치부터 실전 패턴까지 간결하게 정리한다.

어떤 클라이언트를 선택해야 할지 고민된다면 아래 기준을 참고한다.

| 상황 | 권장 클라이언트 |
|------|----------------|
| JVM 기반 백엔드, 기존 JDBC 연동 | Java JDBC |
| Node.js 서버 또는 API 서버 | Node.js (Neo API) |
| Go 마이크로서비스 | Go (`database/sql`) |
| 시스템 프로그래밍, 성능 최우선 | Rust (`duckdb` crate) |
| 통계 분석, ggplot2·dplyr 연동 | R |
| Excel·Power BI·Tableau 연동 | ODBC |
| 대용량 Arrow 기반 데이터 파이프라인 | ADBC |
| 서버 없이 브라우저에서 분석 | WebAssembly (WASM) |

```java
// Java JDBC — 기본 연결 예시
Class.forName("org.duckdb.DuckDBDriver");
Connection conn = DriverManager.getConnection("jdbc:duckdb:");

try (Statement stmt = conn.createStatement();
     ResultSet rs = stmt.executeQuery(
        "SELECT dept, AVG(salary) AS avg_sal FROM employees GROUP BY dept")) {
    while (rs.next()) {
        System.out.printf("%-15s%.2f%n",
            rs.getString("dept"),
            rs.getDouble("avg_sal"));
    }
}
```

## 주요 학습 내용

- Java 환경에서는 HikariCP 같은 연결 풀을 사용할 때 `maximumPoolSize=1`로 제한해야 한다. DuckDB는 동시 쓰기 연결을 허용하지 않으므로 풀 크기를 늘리면 잠금 오류가 발생한다.
- ADBC(`adbc-driver-duckdb`)는 Arrow 기반의 고성능 데이터 전송 인터페이스다. `cursor.fetch_arrow_table()`로 결과를 Arrow 포맷으로 직접 수신해 다른 Arrow 호환 시스템과 Zero-Copy 연동이 가능하다.
- WebAssembly 버전(`@duckdb/duckdb-wasm`)은 브라우저 안에서 DuckDB를 실행한다. 서버 없이 클라이언트 사이드에서 대용량 Parquet 파일을 Range 요청으로 부분 다운로드해 분석할 수 있다.
