# 10-2. 스키마 통합

여러 파일을 합칠 때 스키마가 정확히 일치하지 않는 경우가 많다. 열이 추가되거나, 타입이 바뀌거나, 열 순서가 다를 수 있다.

> **주의:** 기본 동작은 위치 기반 병합이므로, 파일 간 열 순서가 다르면 데이터가 잘못된 열에 들어가는 조용한 오류(silent corruption)가 발생할 수 있다. 파일 출처가 다양하다면 항상 `union_by_name=true`를 명시하는 것이 안전하다.

### 기본: 위치 기반 vs 이름 기반 병합

```sql
-- 기본 동작: 위치 기반 병합 (열 순서가 같아야 함)
SELECT * FROM read_parquet(['a.parquet', 'b.parquet']);

-- union_by_name: 열 이름 기반 병합 (순서 달라도 OK)
SELECT * FROM read_parquet(
    ['a.parquet', 'b.parquet'],
    union_by_name = true
);
```

```
위치 기반 vs 이름 기반 비교
──────────────────────────────────────────────────────────
파일 A: id INTEGER, name VARCHAR, salary DOUBLE
파일 B: id INTEGER, salary DOUBLE, name VARCHAR  ← 순서 다름

위치 기반:
  결과 열: id, name, salary
  파일 B의 salary가 name 열로 들어감! (데이터 오염)

이름 기반 (union_by_name=true):
  결과 열: id, name, salary
  열 이름으로 정확히 매칭 → 안전
```

### 열이 추가된 경우

```sql
-- 파일 A: id, name, amount        (3열)
-- 파일 B: id, name, amount, region (4열 — region 추가)

-- union_by_name 없이: 오류 발생
SELECT * FROM read_csv(['a.csv', 'b.csv']);
-- Error: column count mismatch

-- union_by_name=true: 없는 열은 NULL로 채움
SELECT * FROM read_csv(
    ['a.csv', 'b.csv'],
    union_by_name = true
);
-- id | name  | amount | region
-- ───┼───────┼────────┼────────
-- 1  | Alice | 15000  | NULL    ← 파일 A: region 없음
-- 2  | Bob   | 22000  | NULL
-- 3  | Carol | 18000  | Seoul   ← 파일 B: region 있음
-- 4  | David | 31000  | Busan
```

### 타입 충돌 해결

```sql
-- 파일 A: amount DOUBLE
-- 파일 B: amount VARCHAR (잘못 저장된 경우)

-- 자동 타입 업캐스팅 시도 후 실패하면 VARCHAR로 폴백
SELECT * FROM read_parquet(['a.parquet', 'b.parquet'], union_by_name=true);

-- 명시적 타입 지정으로 해결
SELECT * FROM read_csv(
    ['a.csv', 'b.csv'],
    union_by_name = true,
    types = {'amount': 'DOUBLE'}  -- 강제로 DOUBLE로 읽기
);

-- 또는 all_varchar로 읽은 뒤 수동 변환
SELECT
    id::INTEGER,
    name,
    TRY_CAST(amount AS DOUBLE) AS amount,
    region
FROM read_csv(['a.csv', 'b.csv'], all_varchar=true, union_by_name=true);
```

### 진화하는 스키마 처리

시간이 지나며 스키마가 변경된 파일들을 통합하는 실전 패턴이다.

```sql
-- 시나리오: 월별 파일, 시간에 따라 열이 추가됨
-- 2024_01.parquet: id, ts, event, user_id
-- 2024_06.parquet: id, ts, event, user_id, session_id       (session_id 추가)
-- 2024_12.parquet: id, ts, event, user_id, session_id, ab_test (ab_test 추가)

-- 모두 통합 (없는 열은 NULL)
CREATE TABLE all_events AS
SELECT *
FROM read_parquet('events/2024_*.parquet', union_by_name=true);

-- NULL인 열의 채움 비율 확인 (스키마 진화 히스토리 파악)
SELECT
    STRFTIME(ts::DATE, '%Y-%m') AS month,
    COUNT(*) AS total,
    COUNT(session_id) AS has_session,
    COUNT(ab_test)    AS has_ab_test,
    ROUND(COUNT(session_id) * 100.0 / COUNT(*), 1) AS session_pct
FROM all_events
GROUP BY month
ORDER BY month;
-- month   | total | has_session | has_ab_test | session_pct
-- ────────┼───────┼─────────────┼─────────────┼────────────
-- 2024-01 | 50000 | 0           | 0           | 0.0
-- 2024-06 | 62000 | 62000       | 0           | 100.0
-- 2024-12 | 71000 | 71000       | 71000       | 100.0
```

### UNION ALL vs read_parquet 통합 비교

```sql
-- 방법 1: UNION ALL (열이 완전히 일치할 때)
SELECT * FROM 'jan.parquet'
UNION ALL
SELECT * FROM 'feb.parquet'
UNION ALL
SELECT * FROM 'mar.parquet';

-- 방법 2: read_parquet 와일드카드 (권장, 파일 수에 무관)
SELECT * FROM 'data/2024_*.parquet';

-- 방법 3: union_by_name (스키마 불일치 시)
SELECT * FROM read_parquet('data/*.parquet', union_by_name=true);

-- 방법 4: 명시적 정규화 후 UNION ALL (타입 변환 필요 시)
WITH normalized AS (
    SELECT id, ts, amount::DOUBLE AS amount, NULL::VARCHAR AS region
    FROM 'old_format.parquet'
    UNION ALL
    SELECT id, ts, amount, region
    FROM 'new_format.parquet'
)
SELECT * FROM normalized;
```

### 스키마 검증

```sql
-- 파일 간 스키마 일치 여부 확인
SELECT
    file_name,
    name    AS column_name,
    type    AS column_type
FROM parquet_schema('data/*.parquet')
ORDER BY name, file_name;

-- 스키마가 다른 파일 탐지
WITH schemas AS (
    SELECT
        file_name,
        STRING_AGG(name || ':' || type, ',' ORDER BY name) AS schema_sig
    FROM parquet_schema('data/*.parquet')
    GROUP BY file_name
)
SELECT schema_sig, LIST(file_name) AS files, COUNT(*) AS file_count
FROM schemas
GROUP BY schema_sig
ORDER BY file_count DESC;
-- schema_sig                    | files                | file_count
-- ──────────────────────────────┼──────────────────────┼───────────
-- amount:DOUBLE,id:INT32,...    | ['jan.pq','feb.pq'…] | 10
-- amount:FLOAT,id:INT32,...     | ['old_jan.pq']       | 1   ← 타입 다름
```
