# 02-2. 영구 모드 (Persistent Mode)

데이터가 분석 세션을 넘어 계속 남아 있어야 할 때 영구 모드를 사용한다. 영구 모드는 데이터베이스 파일 경로를 지정해 연결을 만드는 방식이다. 데이터는 디스크의 파일에 저장되며, 프로세스를 종료하고 다시 열어도 그대로 유지된다.

### 기본 사용법

**CLI**

```bash
# 새 파일 생성 또는 기존 파일 열기
duckdb my_database.duckdb
```

**Python**

```python
import duckdb

# 연결 생성 (파일이 없으면 자동 생성)
con = duckdb.connect("my_database.duckdb")

con.sql("CREATE TABLE users (id INTEGER, name VARCHAR)")
con.sql("INSERT INTO users VALUES (1, '홍길동'), (2, '이순신')")
con.sql("SELECT * FROM users").show()

con.close()
```

`close()`를 호출하면 DuckDB는 자동으로 **체크포인트(checkpoint)** 를 수행해 변경 사항을 파일에 반영한다. 체크포인트란 메모리의 변경 내용을 디스크 파일과 동기화하는 과정이다. `with` 구문을 쓰면 `close()`를 명시적으로 호출하지 않아도 DuckDB가 블록 종료 시 자동으로 체크포인트를 처리한다.

> **팁:** 프로덕션 코드에서는 `with duckdb.connect(...) as con:` 패턴을 권장한다. 예외가 발생해도 연결이 안전하게 닫히고 데이터가 파일에 기록된다.

```python
with duckdb.connect("my_database.duckdb") as con:
    con.sql("SELECT * FROM users").show()
# with 블록을 벗어나면 자동으로 close() 호출
```

영구 모드 파일을 명명할 때는 확장자 관례도 알아 두면 협업 시 혼란을 줄일 수 있다.

---

### 파일 확장자: `.duckdb` / `.db` / `.ddb`

DuckDB 파일의 확장자에는 제한이 없다. 기술적으로 어떤 확장자든 사용할 수 있다.

```bash
duckdb analysis.duckdb   # 가장 명시적, 권장
duckdb data.db           # SQLite와 혼동될 수 있음
duckdb warehouse.ddd     # 임의 확장자도 가능
```

관례적으로 자주 쓰이는 확장자는 세 가지다.

| 확장자 | 특징 |
|---|---|
| `.duckdb` | DuckDB임을 가장 명확히 표현, **권장** |
| `.db` | 범용적이지만 SQLite 파일과 혼동 가능 |
| `.ddd` | 일부 커뮤니티에서 사용 |

팀 프로젝트나 파이프라인에서는 `.duckdb`를 사용해 파일의 정체를 명확히 하는 것이 좋다.

> **팁:** 기존에 SQLite를 사용한 팀이라면 `.db` 확장자가 익숙할 수 있다. 하지만 DuckDB 파일과 SQLite 파일은 내부 포맷이 완전히 다르므로, `.duckdb`로 구분하면 실수를 방지할 수 있다.

영구 모드를 장기간 운용할 때 또 하나 중요한 주제가 버전 호환성이다.

---

### 하위 호환성 (v0.10~)

DuckDB는 **v0.10부터 스토리지 포맷의 하위 호환성을 공식 보장**한다. 이 말은, 이전 버전으로 만든 파일을 최신 버전의 DuckDB로 열 수 있다는 뜻이다.

예를 들어:

- DuckDB v0.9로 만든 `.duckdb` 파일 → v1.0, v1.1, v1.5 등에서 그대로 읽기 가능
- 버전 업그레이드 시 파일 변환이나 마이그레이션 작업 불필요

```
v0.9 파일 ──→ v0.10 ──→ v1.0 ──→ v1.5.0
                ↑
          하위 호환성 보장 시작
```

단, **역방향 호환성은 보장되지 않는다.** 최신 버전 DuckDB로 저장한 파일을 구버전 DuckDB로 여는 것은 지원하지 않는다.

> **주의:** 특정 새 기능이나 스토리지 최적화를 사용한 파일은 해당 기능을 지원하는 버전 이상에서만 열 수 있다. 팀 내 DuckDB 버전을 통일하거나, 최소 버전 요건을 문서화해 두는 것이 좋다.

영구 모드와 쌍을 이루는 인메모리 모드는 빠른 탐색과 일회성 분석에 특히 강력하다.
