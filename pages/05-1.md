# 05-1. 집계 함수

데이터 분석의 출발점은 "이 집합에서 무엇을 요약할 수 있는가"라는 질문이다. 집계 함수는 여러 행을 하나의 값으로 줄여 그 질문에 답한다. GROUP BY와 함께 쓰거나 전체 결과셋에 적용하며, DuckDB는 표준 SQL 집계 외에도 통계·근사·리스트 집계까지 폭넓게 지원한다.

### 기본 집계

가장 자주 쓰이는 COUNT, SUM, AVG, MIN, MAX는 SQL 표준 집계의 핵심이다. DuckDB는 `FILTER` 절을 지원해, CASE WHEN을 남발하지 않고도 조건별 집계를 한 쿼리에 깔끔하게 담을 수 있다.

```sql
-- 기본 통계
SELECT
    COUNT(*)           AS total_rows,      -- NULL 포함 전체 행 수
    COUNT(salary)      AS non_null_count,  -- NULL 제외
    COUNT(DISTINCT dept) AS dept_count,    -- 고유값 수
    SUM(salary)        AS total_salary,
    AVG(salary)        AS avg_salary,
    MIN(salary)        AS min_salary,
    MAX(salary)        AS max_salary
FROM employees;

-- NULL 처리: AVG는 NULL을 자동으로 제외하고 계산하므로 별도 필터 없이 안전하게 사용할 수 있다
SELECT AVG(score) FROM results;

-- 조건부 집계: FILTER — 여러 조건의 집계를 하나의 SELECT로 처리한다
SELECT
    COUNT(*) FILTER (WHERE dept = 'Engineering') AS eng_count,
    AVG(salary) FILTER (WHERE active = true)     AS active_avg,
    SUM(amount) FILTER (WHERE year = 2024)       AS sales_2024
FROM employees;
```

> **팁:** `FILTER (WHERE ...)` 절은 CASE WHEN보다 가독성이 높고, 옵티마이저가 더 효율적으로 처리한다. 조건부 집계가 필요할 때 우선 고려한다.

### 통계 집계

단순 합계와 평균을 넘어 데이터의 분포를 파악하려면 표준편차, 분위수, 상관계수 같은 통계 집계가 필요하다. DuckDB는 이를 모두 SQL 함수로 제공하므로, 외부 통계 라이브러리 없이도 분포 분석이 가능하다.

```sql
SELECT
    STDDEV(salary)          AS stddev,       -- 표본 표준편차
    STDDEV_POP(salary)      AS stddev_pop,   -- 모 표준편차
    VARIANCE(salary)        AS variance,     -- 표본 분산
    VAR_POP(salary)         AS var_pop,      -- 모 분산
    MEDIAN(salary)          AS median,       -- 중앙값
    MODE(department)        AS mode_dept,    -- 최빈값
    QUANTILE_CONT(salary, 0.25) AS q1,       -- 연속 분위수 (선형 보간)
    QUANTILE_CONT(salary, 0.75) AS q3,
    QUANTILE_DISC(salary, 0.5)  AS median2,  -- 이산 분위수 (실제 값)
    CORR(salary, years_exp)     AS correlation,  -- 피어슨 상관계수
    COVAR_SAMP(salary, bonus)   AS covariance    -- 표본 공분산
FROM employees;

-- 다중 분위수 한 번에
SELECT QUANTILE_CONT(salary, [0.1, 0.25, 0.5, 0.75, 0.9]) AS percentiles
FROM employees;
-- [p10, p25, p50, p75, p90] 배열로 반환
```

> **팁:** `QUANTILE_CONT`는 선형 보간으로 연속적인 분위수를 반환하고, `QUANTILE_DISC`는 실제 데이터에 존재하는 값을 반환한다. 급여·가격처럼 이산적인 값을 다룰 때는 `QUANTILE_DISC`가 더 직관적이다.

### 문자열 집계

여러 행의 문자열을 하나로 합쳐야 할 때, 혹은 그룹별 값을 배열로 수집해야 할 때 문자열 집계를 사용한다. DuckDB는 `STRING_AGG`로 구분자 연결을, `LIST`로 배열 수집을 지원한다.

```sql
SELECT
    -- 구분자로 연결
    STRING_AGG(name, ', ' ORDER BY name) AS name_list,
    -- 리스트로 수집
    LIST(name ORDER BY salary DESC)      AS ranked_names,
    -- 배열로 수집 (LIST와 동일)
    ARRAY_AGG(name)                      AS name_array
FROM employees
GROUP BY department;

-- 예시 결과
-- dept        | name_list              | ranked_names
-- ────────────┼────────────────────────┼────────────────────────
-- Engineering | Alice, Bob, Carol      | ['Carol', 'Bob', 'Alice']
```

### 불리언 집계

```sql
SELECT
    BOOL_AND(is_active)    AS all_active,   -- 모두 true이면 true
    BOOL_OR(is_premium)    AS any_premium,  -- 하나라도 true이면 true
    EVERY(is_verified)     AS all_verified  -- BOOL_AND와 동일
FROM users
GROUP BY region;
```

### 비트 집계

```sql
SELECT
    BIT_AND(flags) AS common_flags,   -- 비트 AND
    BIT_OR(flags)  AS any_flag,       -- 비트 OR
    BIT_XOR(flags) AS xor_flags       -- 비트 XOR
FROM permissions
GROUP BY role;
```

### 근사 집계 (대용량 최적화)

수억 건 이상의 대용량 데이터에서 정확한 `COUNT(DISTINCT)`는 메모리와 시간을 많이 소비한다. DuckDB가 제공하는 근사 집계 함수는 오차 1% 수준으로 훨씬 빠르게 결과를 반환하므로, 탐색적 분석 단계에서 유용하다.

```sql
-- 정확한 COUNT(DISTINCT)는 비용이 큼
-- HyperLogLog로 근사 카디널리티 (오차 ~1%)
SELECT APPROX_COUNT_DISTINCT(user_id) AS approx_users
FROM events;

-- 근사 분위수 (T-Digest 알고리즘): 수억 건 로그에서 p99 응답시간을 빠르게 추출
SELECT APPROX_QUANTILE(response_time, 0.99) AS p99
FROM api_logs;
```

> **주의:** 근사 집계는 매 실행마다 결과가 미세하게 달라질 수 있다. 정밀도가 중요한 보고서나 과금 집계에는 정확한 집계 함수를 사용한다.

### FIRST / LAST / ARG\_MIN / ARG\_MAX

그룹 안에서 "가장 먼저 입사한 사람의 이름"이나 "급여가 가장 낮은 직원"처럼 다른 열의 순서에 따라 값을 선택해야 할 때, 서브쿼리 없이 `ARG_MIN`·`ARG_MAX`로 간결하게 표현한다.

```sql
SELECT
    -- 그룹의 첫 번째/마지막 값
    FIRST(name ORDER BY hire_date) AS first_hired,
    LAST(name ORDER BY hire_date)  AS last_hired,
    -- 특정 값이 최소/최대일 때 다른 열의 값
    ARG_MIN(name, salary)   AS lowest_paid_name,   -- salary 최솟값의 name
    ARG_MAX(name, salary)   AS highest_paid_name   -- salary 최댓값의 name
FROM employees
GROUP BY department;
```

집계 함수로 데이터를 수치로 요약했다면, 이제 그 수치를 만들어내는 원재료인 문자열을 다뤄보자.
