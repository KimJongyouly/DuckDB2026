# 01-1. DuckDB 설치

DuckDB는 크게 두 가지 방식으로 설치할 수 있다. 하나는 터미널에서 바로 사용하는 **CLI(Command Line Interface)** 방식이고, 다른 하나는 각 프로그래밍 언어의 패키지 매니저를 통해 **라이브러리**로 설치하는 방식이다. 어떤 방식을 선택하든 설치 과정은 단 한 줄 명령으로 끝나며, 설치 직후부터 데이터를 분석할 수 있다.

> **주의:** 이 책의 예제는 **DuckDB v1.5.0 (Current)** 기준으로 작성되었다.
> 안정성이 중요한 프로덕션 환경에서는 LTS 버전(v1.4.4)을 고려할 수 있다. 버전에 따라 일부 기능이나 SQL 문법이 다를 수 있으므로, 설치 후 반드시 버전을 확인한다.

---

### CLI 설치

CLI는 단일 실행 파일이다. 의존성이 없고, 설치 후 바로 쓸 수 있다. PostgreSQL이나 MySQL처럼 서버 프로세스를 먼저 실행할 필요가 없기 때문에, 명령 하나로 DuckDB가 즉시 실행 가능한 상태가 된다.

**macOS**

macOS에서는 설치 스크립트와 Homebrew 두 가지 방법을 모두 지원한다. 설치 스크립트는 최신 버전을 자동으로 내려받아 `/usr/local/bin`에 배치하고, Homebrew는 `brew upgrade duckdb`로 버전 관리가 편리하다.

```bash
# 권장: 설치 스크립트 (최신 버전 자동 설치)
curl https://install.duckdb.org | sh

# Homebrew를 사용하는 경우
brew install duckdb
```

> **팁:** Apple Silicon(M1/M2/M3) Mac이라면 설치 스크립트가 자동으로 ARM64 바이너리를 선택한다. 별도로 아키텍처를 지정할 필요가 없다.

**Linux**

Linux에서도 설치 스크립트로 한 줄에 설치할 수 있다. 스크립트가 현재 시스템의 아키텍처를 감지하여 알맞은 바이너리를 내려받아 설치한다.

```bash
# 설치 스크립트
curl https://install.duckdb.org | sh
```

설치 스크립트를 사용할 수 없는 환경이라면 아키텍처에 맞는 바이너리를 직접 내려받을 수도 있다.

- x86_64: `duckdb_cli-linux-amd64.zip`
- ARM64: `duckdb_cli-linux-arm64.zip`

> **팁:** curl이 설치되지 않은 최소 환경에서는 GitHub Releases 페이지(`https://github.com/duckdb/duckdb/releases`)에서 직접 바이너리를 내려받아 실행 권한(`chmod +x duckdb`)을 부여하면 바로 사용할 수 있다.

**Windows**

Windows에서는 winget 패키지 매니저를 권장한다. PowerShell 스크립트 방식도 동작하지만, winget은 이후 `winget upgrade DuckDB.cli`로 업데이트까지 한 줄로 처리할 수 있다.

```powershell
# winget (권장)
winget install DuckDB.cli

# PowerShell 스크립트
powershell -NoExit iex (iwr "https://install.duckdb.org/install.ps1").Content
```

> **주의:** Windows에서 PowerShell 스크립트 실행 시 "이 시스템에서 스크립트를 실행할 수 없습니다" 오류가 나타날 수 있다. 이 경우 PowerShell을 관리자 권한으로 열고 `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` 명령으로 실행 정책을 변경한 뒤 다시 시도한다.

**Docker (플랫폼 무관)**

Docker를 사용하면 OS나 아키텍처에 관계없이 동일한 환경에서 DuckDB를 실행할 수 있다. 현재 디렉토리를 컨테이너 내부의 `/workspace`에 마운트하므로, 로컬 파일을 컨테이너 안에서 바로 쿼리할 수 있다.

```bash
docker run --rm -it -v "$(pwd):/workspace" -w /workspace duckdb/duckdb
```

설치 후 다음 명령으로 버전을 확인한다. 출력된 버전이 예상과 다를 경우, 시스템에 이전 버전이 남아 있을 수 있으므로 `which duckdb`로 실행 파일 위치를 점검한다.

```bash
duckdb --version
# v1.5.0 ...
```

---

CLI 설치가 끝났다면, 이제 언어별 라이브러리 설치로 넘어간다. 각 언어의 패키지 매니저를 통해 DuckDB를 프로젝트에 통합하면, 기존 코드베이스 안에서 SQL로 데이터를 처리할 수 있다.

### Python

데이터 분석 환경에서 DuckDB를 가장 많이 쓰는 방식이다. Python의 `duckdb` 패키지는 pandas, Polars, PyArrow와 같은 데이터 프레임 라이브러리와 직접 연동되어, 결과를 데이터 프레임으로 받아오거나 데이터 프레임을 테이블처럼 쿼리할 수 있다.

```bash
pip install duckdb
```

가상환경(venv, conda)을 사용하고 있다면 활성화한 상태에서 실행한다.

```bash
# 설치 확인
python -c "import duckdb; print(duckdb.__version__)"
# 1.5.0
```

> **팁:** Jupyter Notebook 환경에서는 `%pip install duckdb`로 설치할 수 있다. DuckDB는 pandas나 NumPy와 달리 C 컴파일러가 필요 없는 바이너리 패키지로 배포되므로, 어떤 환경에서도 설치가 빠르게 완료된다.

---

### Java (JDBC)

DuckDB는 표준 JDBC 드라이버를 제공하기 때문에, 기존 Java 애플리케이션에서 `java.sql.Connection` 인터페이스를 그대로 사용할 수 있다. 별도의 서버를 구성할 필요 없이 JDBC URL만 변경하면 DuckDB로 전환된다. Maven이나 Gradle 프로젝트에서 의존성을 추가한다.

**Maven (`pom.xml`)**

```xml
<dependency>
    <groupId>org.duckdb</groupId>
    <artifactId>duckdb_jdbc</artifactId>
    <version>1.5.0.0</version>
</dependency>
```

**Gradle (`build.gradle`)**

```groovy
implementation("org.duckdb:duckdb_jdbc:1.5.0.0")
```

---

### Node.js

Node.js 환경에서 DuckDB를 사용하면 백엔드 서버가 직접 대용량 데이터를 분석하고 그 결과를 API로 반환할 수 있다. `@duckdb/node-api` 패키지는 Promise 기반의 비동기 API를 제공하므로, 기존 async/await 패턴과 자연스럽게 통합된다.

```bash
npm install @duckdb/node-api
```

> **팁:** 구버전 `duckdb` 패키지(npm)와 혼동하지 않도록 주의한다. `@duckdb/node-api`가 공식 패키지이며, 더 나은 성능과 최신 기능을 지원한다.

---

### 기타 언어

DuckDB는 위 언어 외에도 R, Rust, Go에서도 공식 패키지를 제공한다. 통계 분석에 특화된 R, 시스템 프로그래밍 언어인 Rust, 서버 개발에 주로 쓰이는 Go 모두 동일한 DuckDB 엔진을 내장 라이브러리 형태로 사용한다.

| 언어 | 설치 명령 |
|---|---|
| R | `install.packages("duckdb")` |
| Rust | `cargo add duckdb --features bundled` |
| Go | `go get github.com/duckdb/duckdb-go/v2` |

설치 방법을 살펴봤으니, 이제 CLI를 직접 실행해 첫 쿼리를 입력해본다.
