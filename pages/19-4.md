# 19-4. 제한(Limits)과 트러블슈팅

DuckDB를 운영하다 보면 예상치 못한 한계와 오류를 만난다. 자주 발생하는 문제의 원인과 해결책을 정리한다.

### 주요 제한 사항

```
┌─────────────────────────────────┬───────────────────┬──────────────────────────────────────┐
│ 제한 항목                        │ 한계값             │ 우회 방법                             │
├─────────────────────────────────┼───────────────────┼──────────────────────────────────────┤
│ 동시 쓰기 연결                   │ 1개               │ 단일 쓰기 프로세스 + 읽기 전용 복수   │
│ 데이터베이스 파일 크기            │ 이론상 무제한      │ 실제 운영은 수백 GB 내외 권장         │
│ 컬럼 수                         │ 1,048,576개       │ 정규화 또는 STRUCT 타입으로 묶기      │
│ VARCHAR 길이                    │ 4GB               │ BLOB 고려                            │
│ LIST 중첩 깊이                  │ 100단계            │ 데이터 모델 재설계                    │
│ 표현식 중첩 깊이                 │ 1000 (설정 가능)   │ SET max_expression_depth = 2000      │
│ LIMIT 최댓값                   │ BIGINT MAX        │ 제한 없음                             │
│ 동시 읽기 연결                   │ 제한 없음          │ -                                    │
└─────────────────────────────────┴───────────────────┴──────────────────────────────────────┘
```

**파일을 찾을 수 없음**

```
Error: IO Error: No files found that match the pattern
       "s3://bucket/data/*.parquet"
```

```python
# 원인: 경로 오타, 권한 없음, 파일 없음, Secret 미설정
# 진단 단계

# 1. Secret 확인
con.execute("FROM duckdb_secrets()").fetchdf()

# 2. S3 버킷 내 파일 목록
con.execute("SELECT * FROM glob('s3://bucket/data/*.parquet')").fetchall()

# 3. 연결 테스트
con.execute("SELECT 1 FROM read_parquet('s3://bucket/data/single_file.parquet') LIMIT 1")

# 4. httpfs 확장 로드 확인
con.execute("LOAD httpfs")
```

**스키마 불일치 (union_by_name 없이 다중 파일)**

```
Error: STRUCT equality is only supported for structs with the same key set
```

```sql
-- 원인: 파일마다 컬럼 구성이 다름
-- 해결: union_by_name=true로 스키마 병합
SELECT *
FROM read_parquet('/data/logs/**/*.parquet', union_by_name=true);

-- 누락 컬럼은 NULL로 채워짐
```

**표현식 깊이 초과**

```
Error: Max expression depth limit of 1000 exceeded
```

```sql
-- 원인: 깊게 중첩된 CTE나 서브쿼리

-- 또는 쿼리를 여러 단계로 분리 (임시 테이블 활용)

CREATE TEMP TABLE step2 AS SELECT ... FROM step1;
SELECT ... FROM step2;
```

**타입 변환 오류**

```
Error: Conversion Error: Could not convert string '2024/01/15' to DATE
```

```sql
-- 원인: 날짜 형식 불일치
-- 해결 1: STRPTIME으로 형식 명시
SELECT STRPTIME(date_col, '%Y/%m/%d')::DATE FROM raw_data;

-- 해결 2: TRY_CAST로 안전한 변환 (실패 시 NULL)
SELECT TRY_CAST(date_col AS DATE) FROM raw_data;

-- 해결 3: CSV 읽기 시 dateformat 옵션
SELECT * FROM read_csv('data.csv', dateformat='%Y/%m/%d');
```

### 체크포인트 및 복구

```sql
-- 정상 종료 전 강제 체크포인트 (WAL → 메인 파일 병합)
CHECKPOINT;
CHECKPOINT FORCE;    -- 변경 사항 없어도 강제 실행

-- WAL 파일 크기 확인
-- analytics.db.wal 파일이 클수록 복구 시간이 길어짐
-- 주기적 CHECKPOINT로 WAL 크기 관리
```

```python
# 비정상 종료 후 복구 확인
import duckdb
import os

db_path = '/data/analytics.db'
wal_path = f'{db_path}.wal'

if os.path.exists(wal_path):
    wal_size = os.path.getsize(wal_path)
    print(f"WAL 파일 발견: {wal_size / 1e6:.1f}MB - 복구 중...")

# DuckDB는 재연결 시 WAL을 자동으로 재적용
con = duckdb.connect(db_path)
print("복구 완료")

# 복구 후 체크포인트로 WAL 제거
con.execute("CHECKPOINT")
print(f"WAL 정리 완료: {os.path.exists(wal_path)}")
```

### 성능 진단 플로차트

```
쿼리가 느리다
     │
     ▼
EXPLAIN ANALYZE 실행
     │
     ├─ 추정 행 수 << 실제 행 수 ?──→ ANALYZE 실행 후 재시도
     │
     ├─ SEQ_SCAN인데 작은 테이블 ?──→ 인덱스 생성 고려
     │
     ├─ HASH_JOIN이 느림 ?──────────→ 빌드 측 테이블 필터링 강화
     │
     ├─ SORT가 병목 ?───────────────→ TOP_N 가능 여부 확인 (LIMIT 추가)
     │
     ├─ 스레드가 1개로만 실행 ?──────→ SET threads = N 확인
     │
     └─ 메모리 스필 발생 ?───────────→ memory_limit 증가 또는 배치 분할
```

### 디버깅용 유용한 쿼리 모음

```sql
-- DuckDB 버전 확인
SELECT version();

-- 현재 설정 전체 조회
SELECT name, value, description
FROM duckdb_settings()
WHERE value != default_value   -- 기본값에서 변경된 설정만
ORDER BY name;

-- 실행 중인 쿼리 확인
SELECT * FROM duckdb_tasks();

-- 테이블 크기 추정
SELECT
    table_name,
    estimated_size,
    column_count
FROM duckdb_tables()
ORDER BY estimated_size DESC NULLS LAST;

-- 인덱스 목록
SELECT *
FROM duckdb_indexes()
WHERE database_name = current_database();

-- 확장 상태 확인
SELECT
    extension_name,
    loaded,
    installed,
    install_path
FROM duckdb_extensions()
ORDER BY loaded DESC, extension_name;

-- 메모리 상태 스냅샷
SELECT
    round(memory_usage    / 1e9, 2) AS memory_usage_gb,
    round(memory_limit    / 1e9, 2) AS memory_limit_gb,
    round(memory_usage * 100.0 / memory_limit, 1) AS usage_pct,
    round(temporary_storage_usage / 1e9, 2) AS spill_gb
FROM duckdb_memory();

-- 파일 통계 (Parquet)
SELECT
    file_name,
    num_rows,
    num_row_groups,
    round(total_compressed_size / 1e6, 1) AS compressed_mb,
    round(total_uncompressed_size / 1e6, 1) AS uncompressed_mb
FROM parquet_file_metadata('data.parquet');
```
