# 10-3. Hive 파티셔닝

Hive 파티셔닝은 디렉터리 이름에 `key=value` 형식으로 파티션 정보를 인코딩하는 규약이다. 데이터 레이크에서 사실상 표준으로 자리잡았으며, Spark, Hive, Athena, DuckDB 모두 이 규약을 이해한다.

### Hive 파티션 구조

```
데이터 레이크 예시
──────────────────────────────────────────────────────────
warehouse/
├── year=2023/
│   ├── month=01/
│   │   ├── region=Seoul/
│   │   │   └── 000.parquet   (행: year=2023, month=01, region=Seoul)
│   │   └── region=Busan/
│   │       └── 000.parquet   (행: year=2023, month=01, region=Busan)
│   └── month=02/
│       └── region=Seoul/
│           └── 000.parquet
└── year=2024/
    └── month=01/
        └── region=Seoul/
            └── 000.parquet
```

```
장점
────────────────────────────────────────────────
Partition Pruning: WHERE year=2024 → year=2024/ 만 읽음
병렬 처리: 파티션별 독립 처리 가능
증분 추가: 기존 파티션 건드리지 않고 새 파티션 추가
크로스 도구 호환: Spark, Athena, DuckDB 공유 가능
```

### Hive 파티션 읽기

```sql
-- hive_partitioning=true: 디렉터리명을 열로 자동 파싱
SELECT *
FROM read_parquet('warehouse/**/*.parquet', hive_partitioning=true);

-- 결과: 파티션 열이 자동으로 추가됨
-- year | month | region | id | amount | ...
-- ─────┼───────┼────────┼────┼────────┼───
-- 2023 | 01    | Seoul  |  1 | 15000  |
-- 2023 | 01    | Busan  |  5 |  8500  |

-- 파티션 열 타입 명시 (자동 감지가 VARCHAR로 잡힐 때)
SELECT *
FROM read_parquet(
    'warehouse/**/*.parquet',
    hive_partitioning = true,
    hive_types = {
        'year':  'INTEGER',
        'month': 'INTEGER',
        'region': 'VARCHAR'
    }
);
```

> **팁:** `hive_types`로 파티션 열의 타입을 명시하면 DuckDB가 WHERE 절의 타입 비교를 정확하게 수행한다. 자동 감지에 맡기면 `year`와 `month`가 VARCHAR로 인식되어 `WHERE year > 2023` 같은 범위 조건이 의도와 다르게 동작할 수 있다.

### Partition Pruning (파티션 가지치기)

WHERE 절에 파티션 열이 포함되면 DuckDB는 해당 파티션 디렉터리만 읽는다. 이 동작을 Partition Pruning(파티션 가지치기)이라 하며, 읽어야 할 파일 수를 극적으로 줄여 I/O를 절감한다.

```sql
-- 2024년 데이터만 → year=2024/ 디렉터리만 읽음
SELECT *
FROM read_parquet('warehouse/**/*.parquet', hive_partitioning=true)
WHERE year = 2024;

-- 2024년 1~3월, Seoul만 → 정확히 3개 파티션만 읽음
SELECT *
FROM read_parquet('warehouse/**/*.parquet', hive_partitioning=true)
WHERE year = 2024
  AND month BETWEEN 1 AND 3
  AND region = 'Seoul';

-- EXPLAIN으로 파티션 가지치기 확인
EXPLAIN SELECT *
FROM read_parquet('warehouse/**/*.parquet', hive_partitioning=true)
WHERE year = 2024 AND region = 'Seoul';
-- 출력에서 "Filters: year=2024, region=Seoul" 확인
```

```
Partition Pruning 효과 예시
───────────────────────────────────────────────────────────
전체 파티션:  year(3) × month(12) × region(5) = 180개 파일
쿼리 조건:   year=2024, month IN (1,2,3), region='Seoul'
읽는 파티션: 1 × 3 × 1 = 3개 파일
절감 비율:   180개 → 3개 (98.3% I/O 절감)
```

### 파티션 열 활용 패턴

```sql
-- 파티션 열로 집계 (전체 파일을 읽지 않음)
SELECT year, month, COUNT(*) AS orders, SUM(amount) AS revenue
FROM read_parquet('warehouse/**/*.parquet', hive_partitioning=true)
WHERE year = 2024
GROUP BY year, month
ORDER BY month;

-- 최신 파티션만 읽기
SELECT MAX(year) AS latest_year, MAX(month) AS latest_month
FROM read_parquet('warehouse/**/*.parquet', hive_partitioning=true);

-- 증분 처리: 어제 파티션만 읽기
SELECT *
FROM read_parquet('events/**/*.parquet', hive_partitioning=true)
WHERE year  = YEAR(CURRENT_DATE - INTERVAL '1 day')
  AND month = MONTH(CURRENT_DATE - INTERVAL '1 day')
  AND day   = DAY(CURRENT_DATE - INTERVAL '1 day');

-- 파티션 목록 조회 (glob 활용)
SELECT DISTINCT
    REGEXP_EXTRACT(file, 'year=(\d+)') AS year,
    REGEXP_EXTRACT(file, 'month=(\d+)') AS month
FROM glob('warehouse/**/*.parquet')
ORDER BY year, month;
```

### CSV에서 Hive 파티션 읽기

```sql
SELECT *
FROM read_csv(
    'data/**/*.csv',
    hive_partitioning = true,
    header = true
);

-- JSON도 동일
SELECT *
FROM read_json(
    'logs/**/*.ndjson',
    hive_partitioning = true,
    format = 'newline_delimited'
);
```

> **주의:** Partition Pruning은 WHERE 절 조건이 파티션 열과 정확히 일치할 때만 동작한다. 파티션 열에 함수를 적용하면(`WHERE CAST(year AS VARCHAR) = '2024'` 등) DuckDB가 Pruning을 적용하지 못하고 전체 파일을 스캔한다. 파티션 열은 항상 원본 타입 그대로 비교해야 한다.
