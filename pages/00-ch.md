# 들어가며

데이터 분석의 세계는 빠르게 변하고 있다. 수십 기가바이트, 수백 기가바이트에 이르는 데이터를 노트북 한 대에서 분석해야 하는 일이 점점 더 일상이 되었다. 그런데 막상 도구를 고르려 하면 선택지가 너무 많고, 저마다 장단점이 명확하다. 서버를 띄우기엔 과하고, 그렇다고 단순한 파일 DB로는 부족한 그 어딘가—DuckDB는 바로 그 자리를 정확히 파고든 데이터베이스다.

DuckDB는 2019년 네덜란드 암스테르담 대학교의 Mark Raasveldt와 Hannes Mühleisen이 논문과 함께 공개한 **임베디드 분석 데이터베이스**다. 현재는 DuckDB 재단(DuckDB Foundation)의 관리 아래 MIT 라이선스로 배포되는 완전한 오픈소스 프로젝트이며, Python·R·Go·Rust·Java·Node.js·WebAssembly 등 거의 모든 언어에서 사용할 수 있다.

> **DuckDB는 별도의 서버 없이 프로세스 안에 내장되어 동작하는, OLAP(분석 처리) 전용 SQL 데이터베이스다.**

---

## 주요 학습 내용

- DuckDB는 서버가 필요 없는 임베디드 분석 데이터베이스로, `pip install duckdb` 한 줄로 시작할 수 있다.
- 컬럼 기반 벡터 실행 엔진 덕분에 대용량 집계·조인 쿼리에서 뛰어난 성능을 발휘한다.
- CSV, Parquet, JSON, S3 파일을 테이블에 적재하지 않고 경로만으로 직접 쿼리할 수 있다.
- OLTP(단건 읽기·쓰기)보다 OLAP(대량 분석)에 최적화되어 있으며, 동시 쓰기는 단일 프로세스로 제한된다.
- 데이터 분석가, 데이터 엔지니어, SQL을 배우려는 개발자 모두에게 적합한 도구다.
