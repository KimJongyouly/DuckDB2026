# 02-3. 인메모리 모드 (In-Memory Mode)

파일을 만들지 않고 바로 데이터를 다루고 싶을 때 인메모리 모드를 사용한다. 인메모리 모드는 모든 데이터를 RAM에만 저장한다. 디스크에 어떤 파일도 생성하지 않으며, **프로세스가 종료되면 모든 데이터가 사라진다.**

> **팁:** "데이터가 사라진다"는 점을 단점으로만 볼 필요는 없다. CSV나 Parquet 같은 원본 파일에서 읽어서 분석하고, 결과물만 별도로 저장하는 패턴에서는 중간 DB 파일이 오히려 불필요하다. 인메모리 모드가 이 패턴에 가장 잘 맞는다.

### 기본 사용법

**CLI**

```bash
# 방법 1: 아무 인수 없이 실행
duckdb

# 방법 2: :memory: 명시
duckdb :memory:
```

**Python**

```python
import duckdb

# 방법 1: 인수 없이 연결
con = duckdb.connect()

# 방법 2: :memory: 명시
con = duckdb.connect(":memory:")

# 방법 3: 모듈 레벨 함수 (전역 인메모리 DB)
duckdb.sql("SELECT 42 AS answer").show()
```

방법 3의 `duckdb.sql()`은 Python 모듈 내부에 전역으로 존재하는 인메모리 DB를 사용한다. DuckDB가 내부적으로 전역 연결 객체를 관리하기 때문에 `connect()`를 호출할 필요 없이 바로 SQL을 실행할 수 있다. 빠른 탐색이나 일회성 쿼리에 편리하다.

> **주의:** `duckdb.sql()`의 전역 연결은 멀티스레드 환경에서 공유하면 충돌이 발생할 수 있다. 멀티스레드 코드에서는 스레드마다 `duckdb.connect()`로 별도 연결을 만드는 것이 안전하다.

### 인메모리 모드가 유용한 경우

인메모리 모드는 영속성이 필요 없는 시나리오에 최적이다.

- **탐색적 데이터 분석(EDA):** Jupyter Notebook에서 CSV를 읽고 빠르게 통계를 내는 경우
- **ETL 중간 처리:** 파이프라인 중간 단계에서 변환 작업 후 결과를 다른 시스템으로 넘기는 경우
- **테스트 및 프로토타이핑:** 영구 저장 없이 쿼리 로직을 검증하는 경우
- **일회성 분석:** 분석 결과를 파일로 내보내면 DB 파일 자체는 필요 없는 경우

```python
# 예: CSV를 읽어 집계한 뒤 결과만 저장
import duckdb

result = duckdb.sql("""
    SELECT
        region,
        SUM(sales) AS total_sales
    FROM 'sales_2024.csv'
    GROUP BY region
    ORDER BY total_sales DESC
""").df()  # Pandas DataFrame으로 반환

result.to_csv("summary.csv", index=False)
# DuckDB 인스턴스는 종료되어도 무관 — 결과는 CSV에 저장됨
```

인메모리 모드가 "RAM에만 저장한다"고 해서 처리할 수 있는 데이터 크기가 RAM 크기로 한정된다고 오해하기 쉽다. 다음 섹션에서 그 오해를 바로잡는다.
