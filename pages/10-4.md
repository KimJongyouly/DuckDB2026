# 10-4. 파티션 쓰기

### PARTITION_BY 기본

```sql
-- 단일 열 파티션
COPY (SELECT * FROM sales) TO 'output/' (
    FORMAT       PARQUET,
    PARTITION_BY (region)
);
-- output/region=Seoul/data_0.parquet
-- output/region=Busan/data_0.parquet
-- output/region=Incheon/data_0.parquet

-- 다중 열 파티션 (계층 구조)
COPY (SELECT * FROM sales) TO 'output/' (
    FORMAT       PARQUET,
    PARTITION_BY (year, month, region)
);
-- output/year=2024/month=1/region=Seoul/data_0.parquet
-- output/year=2024/month=1/region=Busan/data_0.parquet
-- ...
```

### 파티션 열 파생

원본 테이블에 파티션 열이 없을 때 파생해서 추가한다.

```sql
-- 날짜에서 year, month, day 파생
COPY (
    SELECT
        *,
        YEAR(order_date)  AS year,
        MONTH(order_date) AS month,
        DAY(order_date)   AS day
    FROM orders
) TO 'orders_partitioned/' (
    FORMAT       PARQUET,
    PARTITION_BY (year, month),
    COMPRESSION  ZSTD
);

-- 파티션 열을 결과에서 제외하고 싶을 때
-- (디렉터리에 있으므로 데이터 중복 없앰)
COPY (
    SELECT
        id, order_date, amount, customer_id,  -- 원본 열 (order_date 유지)
        YEAR(order_date)  AS year,
        MONTH(order_date) AS month
    FROM orders
) TO 'orders_partitioned/' (
    FORMAT       PARQUET,
    PARTITION_BY (year, month),
    COMPRESSION  ZSTD
);
-- 파티션 열(year, month)은 디렉터리명으로 저장
-- 데이터 파일에는 id, order_date, amount, customer_id만 포함
```

### 덮어쓰기 옵션

```sql
-- OVERWRITE_OR_IGNORE: 기존 파티션 덮어씀
COPY (SELECT * FROM sales WHERE year = 2024) TO 'warehouse/' (
    FORMAT              PARQUET,
    PARTITION_BY        (year, month),
    OVERWRITE_OR_IGNORE true
);

-- 기존 파티션을 건드리지 않고 새 파티션만 추가
-- OVERWRITE_OR_IGNORE false (기본): 파일 충돌 시 오류
COPY (SELECT * FROM new_data) TO 'warehouse/' (
    FORMAT       PARQUET,
    PARTITION_BY (year, month)
);
```

### 파일 크기 제어

```sql
-- 파티션당 파일 크기 제어 (DuckDB는 직접 옵션 없음)
-- → Row Group Size + 데이터 크기로 간접 제어

COPY (SELECT * FROM large_sales) TO 'output/' (
    FORMAT          PARQUET,
    PARTITION_BY    (year, month),
    COMPRESSION     ZSTD,
    ROW_GROUP_SIZE  500000    -- Row Group 50만 행
);

-- 파티션당 파일이 너무 많으면 → Python으로 배치 분할
```

```python
import duckdb

con = duckdb.connect('warehouse.duckdb')

# 월별 배치로 나눠서 저장 (파일 크기 제어)
months = con.execute("""
    SELECT DISTINCT year, month
    FROM sales
    ORDER BY year, month
""").fetchall()

for year, month in months:
    con.execute(f"""
        COPY (
            SELECT * FROM sales
            WHERE year = {year} AND month = {month}
        ) TO 'output/year={year}/month={month:02d}/' (
            FORMAT PARQUET,
            COMPRESSION ZSTD,
            ROW_GROUP_SIZE 200000
        )
    """)
    print(f"  year={year}/month={month:02d} 완료")
```

### 증분 파티션 업데이트

기존 데이터를 건드리지 않고 새 데이터를 파티션에 추가하는 패턴이다.

```python
import duckdb
from datetime import date, timedelta

con = duckdb.connect()

def append_daily_partition(target_date: date):
    """하루치 데이터를 파티션에 추가"""
    y, m, d = target_date.year, target_date.month, target_date.day
    output_path = f'warehouse/year={y}/month={m:02d}/day={d:02d}/'

    from pathlib import Path
    if Path(output_path).exists():
        print(f"  이미 존재: {output_path}, 건너뜀")
        return

    con.execute(f"""
        COPY (
            SELECT * EXCLUDE (year, month, day),
                   {y}  AS year,
                   {m}  AS month,
                   {d}  AS day
            FROM source_events
            WHERE event_date = DATE '{target_date}'
        ) TO '{output_path}' (
            FORMAT PARQUET,
            COMPRESSION ZSTD
        )
    """)
    print(f"  작성 완료: {output_path}")

# 최근 7일 파티션 생성
today = date.today()
for i in range(7):
    d = today - timedelta(days=i)
    append_daily_partition(d)
```

### 파티션 설계 가이드

파티션을 어떤 열로, 몇 단계로 나눌지는 성능에 직결된다.

```
파티션 설계 원칙
──────────────────────────────────────────────────────────
1. 쿼리 패턴 기반으로 선택
   → WHERE 절에 자주 등장하는 열이 파티션 후보

2. 카디널리티 균형
   → 너무 낮음: 파티션 효과 없음  (예: boolean)
   → 너무 높음: 파일 수 폭발     (예: user_id)
   → 적정:      수십~수백 개      (예: date, region, category)

3. 파티션당 파일 크기
   → 너무 작음 (<1MB):  파일 수 너무 많음, 오버헤드 증가
   → 너무 큼   (>1GB):  Pruning 효과 감소
   → 적정 범위: 128MB ~ 512MB

4. 파티션 깊이
   → 2~3 단계가 적당 (year/month 또는 year/month/day)
   → 너무 깊으면 디렉터리 탐색 오버헤드
```

```
파티션 설계 예시
──────────────────────────────────────────────────────────
시나리오           권장 파티션
───────────────    ────────────────────────────────
일별 로그 분석     year / month / day
지역별 판매        region / year / month
상품 카탈로그      category / subcategory
멀티테넌트 SaaS    tenant_id / year / month
IoT 센서 데이터    device_type / year / month / day
```

### 파티션 파일 관리

```sql
-- 파티션 목록 조회
SELECT DISTINCT
    REGEXP_EXTRACT(file, 'year=(\d+)') AS year,
    REGEXP_EXTRACT(file, 'month=(\d+)') AS month
FROM glob('warehouse/year=*/month=*/*.parquet')
ORDER BY year, month;

-- 파티션별 파일 수와 크기 (parquet_metadata 활용)
SELECT
    REGEXP_EXTRACT(file_name, 'year=(\d+)') AS year,
    REGEXP_EXTRACT(file_name, 'month=(\d+)') AS month,
    COUNT(DISTINCT file_name) AS file_count,
    SUM(row_group_num_rows)   AS total_rows,
    ROUND(SUM(total_compressed_size) / 1024.0 / 1024.0, 1) AS size_mb
FROM parquet_metadata('warehouse/**/*.parquet')
GROUP BY year, month
ORDER BY year, month;

-- 빈 파티션 또는 너무 작은 파티션 탐지
SELECT
    REGEXP_EXTRACT(file_name, 'year=\d+/month=\d+') AS partition,
    SUM(row_group_num_rows) AS rows,
    ROUND(SUM(total_compressed_size) / 1024.0 / 1024.0, 1) AS size_mb
FROM parquet_metadata('warehouse/**/*.parquet')
GROUP BY partition
HAVING size_mb < 1.0  -- 1MB 미만 소형 파티션
ORDER BY size_mb;
```

### 소형 파일 병합 (Compaction)

```sql
-- 소형 파티션 파일들을 하나로 병합
-- 예: year=2024/month=01/ 의 파일들을 재작성
COPY (
    SELECT *
    FROM read_parquet(
        'warehouse/year=2024/month=01/**/*.parquet',
        hive_partitioning = true
    )
) TO 'warehouse_compact/year=2024/month=01/' (
    FORMAT          PARQUET,
    COMPRESSION     ZSTD,
    ROW_GROUP_SIZE  500000,
    OVERWRITE_OR_IGNORE true
);
```

```python
import duckdb
from pathlib import Path

con = duckdb.connect()

def compact_partition(src_base: str, dst_base: str, year: int, month: int):
    """파티션 소형 파일을 단일 파일로 병합"""
    src = f'{src_base}/year={year}/month={month:02d}/**/*.parquet'
    dst = f'{dst_base}/year={year}/month={month:02d}/'

    # 소스 파일 수 확인
    file_count = len(list(Path(f'{src_base}/year={year}/month={month:02d}')
                          .glob('**/*.parquet')))
    if file_count <= 1:
        print(f"  year={year}/month={month:02d}: 파일 1개, 건너뜀")
        return

    con.execute(f"""
        COPY (SELECT * FROM read_parquet('{src}'))
        TO '{dst}' (
            FORMAT          PARQUET,
            COMPRESSION     ZSTD,
            ROW_GROUP_SIZE  500000,
            OVERWRITE_OR_IGNORE true
        )
    """)
    print(f"  year={year}/month={month:02d}: {file_count}개 → 1개 병합 완료")
```

### 전체 파이프라인 예시

```python
import duckdb
from datetime import date, timedelta

con = duckdb.connect()

def daily_etl_pipeline(process_date: date):
    """일별 ETL: 원본 → 정제 → 파티션 저장"""
    y, m, d = process_date.year, process_date.month, process_date.day

    print(f"\n=== {process_date} ETL 시작 ===")

    # Step 1: 원본 읽기 (여러 소스 통합)
    con.execute(f"""
        CREATE OR REPLACE TEMP TABLE raw AS
        SELECT * FROM read_csv(
            'raw/date={process_date}/*.csv',
            header=true, union_by_name=true, ignore_errors=true
        )
    """)

    # Step 2: 정제
    con.execute("""
        CREATE OR REPLACE TEMP TABLE cleaned AS
        SELECT
            id::INTEGER                             AS id,
            TRY_CAST(ts AS TIMESTAMP)               AS ts,
            NULLIF(TRIM(event_type), '')            AS event_type,
            TRY_CAST(amount AS DECIMAL(12,2))       AS amount,
            LOWER(TRIM(region))                     AS region
        FROM raw
        WHERE TRY_CAST(id AS INTEGER) IS NOT NULL
          AND TRY_CAST(ts AS TIMESTAMP) IS NOT NULL
    """)

    # Step 3: 파티션 저장
    con.execute(f"""
        COPY (
            SELECT *, {y} AS year, {m} AS month, {d} AS day
            FROM cleaned
        ) TO 'warehouse/' (
            FORMAT              PARQUET,
            PARTITION_BY        (year, month, day),
            COMPRESSION         ZSTD,
            OVERWRITE_OR_IGNORE true
        )
    """)

    count = con.execute("SELECT COUNT(*) FROM cleaned").fetchone()[0]
    print(f"  처리 완료: {count:,}행 → warehouse/year={y}/month={m}/day={d}/")

# 최근 3일 재처리
for i in range(3):
    daily_etl_pipeline(date.today() - timedelta(days=i))
```
