# 제18장: 성능 최적화

DuckDB는 기본 설정만으로도 빠르지만, 실행 계획을 이해하고 병렬 처리와 메모리를 올바르게 구성하면 수배에서 수십 배까지 성능을 끌어낼 수 있다. 이 장에서는 DuckDB 쿼리가 실제로 어떻게 실행되는지 들여다보고, 대용량 데이터를 다룰 때 쓸 수 있는 실전 전략을 정리한다.

```sql
-- 성능 최적화의 시작: 실행 계획 확인
EXPLAIN ANALYZE
SELECT region, SUM(amount) AS total
FROM sales
WHERE year = 2024
GROUP BY region
ORDER BY total DESC;
```

## 주요 학습 내용

- EXPLAIN ANALYZE는 아래에서 위로 읽으며, 실측 행 수와 추정치의 괴리가 클 때는 ANALYZE로 통계를 갱신한다.
- 벡터화 실행 엔진 덕분에 DuckDB는 CPU가 한 번에 여러 값을 동시에 처리하며, threads와 memory_limit 설정만으로 이 이점을 최대화할 수 있다.
- 파티셔닝, 배치 처리, 인덱스는 각각 I/O 최소화·메모리 절약·포인트 룩업 가속이라는 서로 다른 문제를 해결하므로, 상황에 맞게 선택해 조합한다.
