# 17-1. 설정(Configuration) 개요

DuckDB의 설정은 세 가지 방법으로 제어할 수 있다: `SET` 명령, 연결 옵션, 그리고 `.duckdbrc` 파일. 설정 값은 세션 단위 또는 글로벌 단위로 적용되며, 일부는 연결 생성 시에만 지정할 수 있다.

### 설정 조회

```sql
-- 전체 설정 목록 조회
SELECT name, value, description
FROM duckdb_settings()
ORDER BY name;

-- 특정 설정 조회
SELECT value FROM duckdb_settings() WHERE name = 'threads';

-- 현재 설정 요약 (PRAGMA 방식)
PRAGMA database_list;
PRAGMA threads;
PRAGMA memory_limit;
```

### SET 명령으로 설정 변경

```sql
-- 스레드 수 설정 (CPU 코어 수에 맞게)
SET threads = 8;

-- 메모리 한도 설정
SET memory_limit = '16GB';

-- 임시 파일 저장 경로 (메모리 초과 시 스필 디렉터리)
SET temp_directory = '/tmp/duckdb_spill';

-- 워킹 디렉터리
SET file_search_path = '/data/csv:/data/parquet';

-- 설정 초기화 (기본값으로 되돌리기)
RESET threads;
RESET memory_limit;
```

### 주요 성능 설정 항목

```
┌─────────────────────────────────┬───────────────────────┬──────────────────────────────────┐
│ 설정 키                          │ 기본값                 │ 설명                              │
├─────────────────────────────────┼───────────────────────┼──────────────────────────────────┤
│ threads                         │ CPU 코어 수            │ 병렬 처리 스레드 수               │
│ memory_limit                    │ RAM의 80%             │ 최대 메모리 사용량                 │
│ temp_directory                  │ OS 임시 디렉터리       │ 메모리 초과 시 스필 경로           │
│ max_expression_depth            │ 1000                  │ 중첩 표현식 최대 깊이              │
│ enable_progress_bar             │ true                  │ 진행 표시줄 활성화                 │
│ enable_profiling                │ false                 │ 쿼리 프로파일링 활성화             │
│ profiling_output                │ (없음)                │ 프로파일 출력 파일 경로            │
│ explain_output                  │ physical_only         │ EXPLAIN 출력 수준                 │
│ preserve_insertion_order        │ true                  │ INSERT 순서 보존                  │
│ default_collation               │ (없음)                │ 기본 문자열 정렬 기준              │
│ default_null_order              │ NULLS_LAST            │ 기본 NULL 정렬 위치               │
│ default_order                   │ ASC                   │ 기본 정렬 방향                    │
│ ieee_floating_point_ops         │ true                  │ IEEE 754 부동소수점 준수           │
│ old_implicit_casting            │ false                 │ 구버전 암묵적 형변환 허용          │
└─────────────────────────────────┴───────────────────────┴──────────────────────────────────┘
```

### 연결 시 설정 (Python)

연결 생성 단계에서만 지정할 수 있는 설정은 `config` 딕셔너리로 전달한다.

```python
import duckdb

# 연결 생성 시 설정 주입
con = duckdb.connect(
    database='analysis.db',
    config={
        'threads': 4,
        'memory_limit': '8GB',
        'temp_directory': '/tmp/duckdb',
        'default_null_order': 'NULLS_LAST',
        'enable_progress_bar': False,   # 배치 환경에서는 비활성화
    }
)

# 연결 후 추가 설정
con.execute("SET enable_profiling = true")
con.execute("SET profiling_output = '/tmp/profile.json'")
```

### .duckdbrc 영구 설정

CLI 세션마다 동일한 설정을 적용하려면 `~/.duckdbrc` 파일에 설정을 저장한다.

```sql
-- ~/.duckdbrc
SET threads = 8;
SET memory_limit = '32GB';
SET temp_directory = '/tmp/duckdb_spill';
SET enable_progress_bar = true;

-- 확장 자동 설치/로드
SET autoinstall_known_extensions = true;
SET autoload_known_extensions = true;

-- 출력 형식
.mode duckbox
.timer on
.headers on
```

### 읽기 전용 모드

```sql
-- CLI에서 읽기 전용으로 열기
duckdb -readonly analysis.db

-- Python에서 읽기 전용 연결
con = duckdb.connect('analysis.db', read_only=True)

-- SQL로 설정 (연결 후에는 변경 불가)

```
