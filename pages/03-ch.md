# 제3장: SQL 기초 및 DuckDB 방언

DuckDB의 SQL은 표준 SQL을 충실히 따르면서도, 데이터 분석에 특화된 편의 문법을 다수 포함한다. PostgreSQL 문법과 높은 호환성을 갖추고 있어 PostgreSQL 경험이 있다면 빠르게 적응할 수 있다. 이 장에서는 SELECT부터 윈도우 함수, 집합 연산까지 DuckDB SQL의 핵심을 차례로 살펴본다. 각 기능이 어떤 문제를 해결하기 위해 존재하는지를 함께 짚으면서, 실무에서 바로 쓸 수 있는 패턴을 익힌다.

```python
import duckdb

con = duckdb.connect()

# .sql() — Relation 객체 반환, 메서드 체이닝 가능
rel = con.sql("SELECT 42 AS answer")
rel.show()

# .execute() — 커서 기반, fetchall() / fetchdf() 등 사용
cur = con.execute("SELECT 42 AS answer")
print(cur.fetchall())
```

---

## 주요 학습 내용

- DuckDB의 `SELECT * EXCLUDE(col)`, `FROM` 절 생략, `COLUMNS()` 패턴 같은 확장 문법은 반복적인 분석 쿼리를 크게 단순화한다.
- 윈도우 함수는 `PARTITION BY`와 `ORDER BY`를 조합해 그룹 내 순위·누계·이동 평균을 계산한다. `QUALIFY`를 사용하면 서브쿼리 없이 윈도우 함수 결과로 행을 필터링할 수 있다.
- CTE(`WITH`)는 복잡한 쿼리를 읽기 쉬운 단계로 분리하고, 재귀 CTE로 계층 구조 데이터를 처리할 수 있다.
