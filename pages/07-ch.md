# 제7장: CSV 파일

CSV(Comma-Separated Values)는 데이터 분석 현장에서 가장 흔하게 마주치는 파일 형식이다. DuckDB는 CSV를 마치 테이블처럼 직접 쿼리할 수 있으며, 자동 타입 감지, 손상 데이터 복구, 다양한 출력 옵션까지 갖추고 있다. 별도의 로드 단계 없이 파일을 SQL로 분석하는 것이 DuckDB CSV 처리의 핵심 철학이다.

```sql
-- 파일을 직접 쿼리 (테이블처럼)
SELECT * FROM 'sales.csv';

-- read_csv_auto: 모든 것을 자동으로 감지
SELECT * FROM read_csv_auto('sales.csv');

-- read_csv: 옵션을 명시적으로 지정
SELECT * FROM read_csv('sales.csv', header=true, delim=',');
```

## 주요 학습 내용

- DuckDB의 CSV 자동 감지는 구분자·타입·헤더를 스스로 추론한다. 하지만 복잡한 파일은 `delimiter`, `types`, `dateformat` 같은 옵션을 명시적으로 지정하는 것이 더 안전하다.
- 한국어 데이터나 레거시 파일에서 EUC-KR 인코딩 문제가 자주 발생한다. `encoding='EUC-KR'`(또는 `CP949`) 옵션으로 해결할 수 있다.
- `COPY (쿼리) TO '파일.csv'`로 SQL 결과를 직접 CSV로 저장한다. `.gz`나 `.zst` 확장자를 붙이면 압축 파일로 자동 출력된다.
