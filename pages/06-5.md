# 06-5. 트랜잭션과 VACUUM / ANALYZE

### 트랜잭션 기초

트랜잭션은 여러 SQL 문을 하나의 논리적 단위로 묶는다. 모두 성공하거나 모두 실패한다(원자성).

```sql
-- 명시적 트랜잭션
BEGIN;
    UPDATE accounts SET balance = balance - 50000 WHERE id = 1;
    UPDATE accounts SET balance = balance + 50000 WHERE id = 2;
COMMIT;

-- 오류 시 롤백
BEGIN;
    DELETE FROM orders WHERE status = 'cancelled';
    INSERT INTO archive_orders SELECT * FROM orders WHERE status = 'cancelled';
    -- 오류 발생 시
ROLLBACK;

-- SAVEPOINT: 부분 롤백
BEGIN;
    INSERT INTO log (msg) VALUES ('작업 시작');

    SAVEPOINT sp1;
    UPDATE inventory SET qty = qty - 10 WHERE product_id = 42;

    SAVEPOINT sp2;
    UPDATE sales SET status = 'shipped' WHERE order_id = 100;

    -- sp2 이후만 롤백
    ROLLBACK TO SAVEPOINT sp2;

    RELEASE SAVEPOINT sp1;  -- 세이브포인트 해제
COMMIT;
```

### DuckDB의 트랜잭션 특성

```
┌─────────────────────────────────────────────────────────────┐
│              DuckDB 트랜잭션 격리 수준                        │
├─────────────────────────────────────────────────────────────┤
│  기본 격리 수준: SERIALIZABLE (직렬화 가능)                   │
│  MVCC 기반: 읽기와 쓰기가 서로 블로킹하지 않음                │
│  단일 쓰기 트랜잭션: 동시에 하나의 쓰기 트랜잭션만 허용        │
│  읽기 트랜잭션: 동시에 여러 개 허용                           │
└─────────────────────────────────────────────────────────────┘
```

```sql
-- 읽기 전용 트랜잭션
BEGIN READ ONLY;
    SELECT * FROM employees;
    SELECT COUNT(*) FROM orders;
COMMIT;

-- 격리 수준 설정 (DuckDB는 SERIALIZABLE만 지원)
BEGIN ISOLATION LEVEL SERIALIZABLE;
    -- ...
COMMIT;

-- 자동 커밋 모드 (기본값)
-- 각 문장이 자동으로 트랜잭션으로 처리됨
UPDATE employees SET salary = 90000 WHERE id = 1;  -- 자동 커밋
```

### Python에서의 트랜잭션

```python
import duckdb

con = duckdb.connect('mydb.duckdb')

# 방법 1: 명시적 BEGIN/COMMIT
con.execute("BEGIN")
try:
    con.execute("UPDATE accounts SET balance = balance - 100 WHERE id = 1")
    con.execute("UPDATE accounts SET balance = balance + 100 WHERE id = 2")
    con.execute("COMMIT")
except Exception as e:
    con.execute("ROLLBACK")
    raise

# 방법 2: Python context manager (권장)
with con.cursor() as cur:
    cur.execute("BEGIN")
    try:
        cur.execute("INSERT INTO orders VALUES (?, ?, ?)", [101, 1, 500])
        cur.execute("UPDATE inventory SET qty = qty - 1 WHERE id = 1")
        cur.execute("COMMIT")
    except:
        cur.execute("ROLLBACK")
        raise
```

### VACUUM: 공간 회수

DuckDB에서 VACUUM은 삭제된 데이터가 차지하는 공간을 회수하고 파일을 최적화한다.

```sql
-- 전체 데이터베이스 정리
VACUUM;

-- 특정 테이블 정리
VACUUM employees;

-- ANALYZE 포함: 통계도 함께 갱신
VACUUM ANALYZE;
VACUUM ANALYZE employees;

-- 언제 실행하는가?
-- - 대량 DELETE / UPDATE 이후
-- - 테이블이 부풀어 오른 경우 (bloat)
-- - 쿼리 성능이 저하된 경우
```

### ANALYZE: 통계 수집

ANALYZE는 쿼리 옵티마이저가 사용하는 테이블 통계를 갱신한다. 데이터가 크게 바뀐 후 실행하면 쿼리 계획이 개선된다.

```sql
-- 전체 통계 갱신
ANALYZE;

-- 특정 테이블
ANALYZE employees;

-- 쿼리 실행 계획 확인
EXPLAIN SELECT * FROM employees WHERE department = 'Engineering';
EXPLAIN ANALYZE SELECT * FROM employees WHERE salary > 80000;

-- 상세 실행 계획 (JSON 형식)
EXPLAIN (FORMAT JSON, ANALYZE TRUE)
SELECT department, AVG(salary)
FROM employees
GROUP BY department;
```

### PRAGMA: 설정 조회 및 변경

```sql
-- 메모리 한도 설정
PRAGMA memory_limit = '4GB';
SET memory_limit = '4GB';

-- 스레드 수 설정
PRAGMA threads = 8;
SET threads = 8;

-- 임시 파일 경로 (디스크 스필링 위치)
SET temp_directory = '/tmp/duckdb';

-- 프로파일링 활성화
PRAGMA enable_profiling;
PRAGMA profiling_output = 'profile.json';

-- 진행률 표시
PRAGMA enable_progress_bar;

-- 현재 설정 조회
SELECT * FROM duckdb_settings();
SELECT current_setting('memory_limit');

-- 데이터베이스 정보
PRAGMA database_list;
CALL pragma_database_size();
SELECT * FROM duckdb_tables();
SELECT * FROM duckdb_columns();
```

### 실전 운영 패턴

```sql
-- 패턴 1: 안전한 대량 업데이트 (배치 처리)
BEGIN;
UPDATE employees
SET salary = salary * 1.05
WHERE department = 'Engineering'
  AND performance_score >= 4.0;

-- 영향받은 행 확인
SELECT changes();  -- 변경된 행 수

COMMIT;

-- 패턴 2: 임시 테이블로 안전하게 작업
CREATE TEMP TABLE employees_update AS
SELECT id, salary * 1.05 AS new_salary
FROM employees
WHERE department = 'Engineering';

-- 검토 후 반영
BEGIN;
UPDATE employees e
SET salary = u.new_salary
FROM employees_update u
WHERE e.id = u.id;
COMMIT;

DROP TABLE employees_update;

-- 패턴 3: 롤링 삭제 (파티션 흉내)
BEGIN;
DELETE FROM events
WHERE event_date < CURRENT_DATE - INTERVAL '90 days';
VACUUM events;
COMMIT;
```
