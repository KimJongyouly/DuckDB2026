# 제19장: 운영 및 보안

DuckDB는 단순한 분석 도구를 넘어 프로덕션 데이터 파이프라인의 핵심 구성 요소로 자리잡고 있다. 컨테이너 기반 배포, 로그와 메트릭 수집, 보안 강화, 그리고 장애 상황의 빠른 진단까지, 운영 환경에서 DuckDB를 안정적으로 다루는 방법을 정리한다.

```dockerfile
# DuckDB는 "DuckDB 서버 컨테이너"가 아닌
# DuckDB를 사용하는 애플리케이션을 컨테이너화
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
ENV DUCKDB_MEMORY_LIMIT=8GB
ENV DUCKDB_THREADS=4
CMD ["python", "main.py"]
```

## 주요 학습 내용

- DuckDB는 임베디드 DB이므로 "DuckDB 서버" 컨테이너는 없다. 애플리케이션을 컨테이너화하고, 쓰기 워커는 반드시 하나로 제한한다.
- SQL 인젝션과 경로 탈출은 실제 위협이다. 파라미터 바인딩과 경로 정규화(`realpath`)를 습관처럼 적용하고, 자격증명은 코드가 아닌 환경 변수나 시크릿 매니저에서 가져온다.
- OOM, 파일 잠금, WAL 복구는 DuckDB 운영에서 가장 자주 마주치는 오류다. 스필 디렉터리 설정과 주기적 `CHECKPOINT`로 대부분의 상황을 사전에 대비할 수 있다.
