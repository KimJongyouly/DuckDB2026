# 제4장: 데이터 타입

데이터 타입은 DuckDB가 값을 저장하고 연산하는 방식의 기반이다. 타입을 정확히 이해하면 불필요한 형변환을 줄이고, 메모리 사용을 최적화하며, 예상치 못한 오류를 방지할 수 있다. DuckDB는 전통적인 스칼라 타입 외에도 LIST, STRUCT, MAP 같은 중첩 타입을 기본으로 지원해 반정형 데이터를 SQL 안에서 자연스럽게 다룰 수 있다.

앞 장에서 데이터를 읽고 쿼리하는 방법을 익혔다면, 이 장에서는 그 데이터가 실제로 어떤 형태로 저장되고 처리되는지를 살펴본다. 올바른 타입을 선택하는 것은 단순한 문법 문제가 아니라 쿼리 성능과 데이터 정확성에 직접 영향을 미치는 설계 결정이다.

---

## 주요 학습 내용

- 재무·정밀 계산에는 `DECIMAL(p,s)`를 사용한다. `FLOAT`과 `DOUBLE`은 부동소수점 오차가 발생할 수 있어 금액 계산에 부적합하다.
- `LIST`, `STRUCT`, `MAP`은 정규화 없이 중첩 데이터를 한 열에 담을 수 있어 JSON 스타일 데이터를 처리할 때 강력하다. `UNNEST`와 조합하면 중첩 데이터를 행으로 전개할 수 있다.
- `ENUM` 타입은 성별, 상태, 등급처럼 고정된 값 목록을 가진 열에 적용하면 문자열 비교 대신 정수 인덱스를 사용하므로 메모리와 성능 모두 효율적이다.
