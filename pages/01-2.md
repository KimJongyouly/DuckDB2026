# 01-2. CLI로 첫 쿼리 실행하기

CLI는 DuckDB를 가장 빠르게 체험할 수 있는 방법이다. 인터랙티브 셸로 들어가 SQL을 바로 실행할 수 있다. DuckDB CLI는 두 가지 실행 모드를 제공하는데, 어떤 모드를 선택하느냐에 따라 데이터가 메모리에만 존재하는지 아니면 파일로 영구 저장되는지가 결정된다.

### CLI 시작하기

**인메모리 모드로 시작** (데이터가 세션 종료와 함께 사라짐)

인메모리 모드는 임시 분석이나 빠른 실험에 적합하다. DuckDB가 모든 데이터를 메모리에 올려 처리하기 때문에 디스크 I/O 없이 최고 속도로 쿼리를 실행한다.

```bash
duckdb
```

**파일 DB로 시작** (데이터가 파일에 영속적으로 저장됨)

파일 DB 모드는 결과를 나중에도 사용해야 할 때 선택한다. DuckDB가 `.duckdb` 확장자의 파일에 스키마와 데이터를 저장하며, 다음에 같은 파일명으로 시작하면 이전 상태가 그대로 복원된다.

```bash
duckdb my_database.duckdb
```

정상적으로 실행되면 다음 프롬프트가 나타난다.

```
v1.5.0 ...
Enter ".help" for usage hints.
D
```

`D` 가 DuckDB CLI의 프롬프트다. 이 프롬프트가 보이면 DuckDB가 SQL을 받아들일 준비가 된 것이다.

---

### 첫 번째 쿼리

```sql
SELECT 'Hello, DuckDB!' AS message;
```

```
┌────────────────┐
│    message     │
│    varchar     │
├────────────────┤
│ Hello, DuckDB! │
└────────────────┘
```

쿼리는 반드시 세미콜론(`;`)으로 끝내야 실행된다. 세미콜론 없이 Enter를 누르면 DuckDB는 쿼리가 계속 이어지는 것으로 판단하고 다음 줄 입력을 기다린다. 덕분에 긴 쿼리를 여러 줄로 나눠 가독성 있게 작성할 수 있다.

> **팁:** 입력 도중 쿼리를 취소하고 싶을 때는 `Ctrl + C`를 누른다. DuckDB가 현재 입력을 버리고 새 프롬프트를 표시한다.

---

### CSV 파일 직접 쿼리하기

DuckDB CLI의 강점을 바로 느낄 수 있는 예제다. 대부분의 데이터베이스는 CSV 파일을 사용하려면 먼저 테이블을 만들고 데이터를 임포트해야 한다. DuckDB는 파일 경로를 테이블처럼 `FROM` 절에 직접 쓸 수 있으며, DuckDB가 CSV 스키마를 자동으로 추론해 바로 쿼리를 실행한다.

```sql
SELECT * FROM 'data.csv' LIMIT 5;
```

파일 경로에 와일드카드도 쓸 수 있다. DuckDB가 패턴에 일치하는 모든 파일을 하나의 테이블처럼 합쳐서 읽어들인다.

```sql
SELECT COUNT(*) FROM 'logs/*.csv';
```

> **팁:** Parquet, JSON, Excel 파일도 같은 방식으로 직접 쿼리할 수 있다. `SELECT * FROM 'data.parquet'`, `SELECT * FROM 'data.json'`처럼 확장자만 바꾸면 DuckDB가 파일 형식을 자동으로 인식한다.

### 자주 쓰는 도트(dot) 명령

SQL 쿼리 외에도 CLI 자체의 동작을 제어하는 전용 명령이 있다. 이를 도트 명령이라 부르며, 점(`.`)으로 시작하고 세미콜론 없이 실행한다. 출력 형식 변경, 다른 DB 파일 열기, SQL 파일 실행 같은 작업을 처리한다.

| 명령 | 설명 |
|---|---|
| `.help` | 전체 명령 목록 보기 |
| `.open [파일]` | 다른 DB 파일로 전환 (`.open :memory:` 로 인메모리 전환) |
| `.mode [형식]` | 출력 형식 변경 (`csv`, `json`, `markdown`, `duckbox`) |
| `.read [파일]` | SQL 파일 실행 |
| `.output [파일]` | 쿼리 결과를 파일로 출력 |
| `.tables` | 현재 DB의 테이블 목록 보기 |
| `.schema [테이블]` | 테이블 구조 보기 |
| `.exit` | CLI 종료 |

출력 형식을 마크다운으로 바꾸면 결과를 문서에 바로 붙여넣을 수 있다.

```
D .mode markdown
D SELECT 42 AS answer;
| answer |
|-------:|
|     42 |
```

### CLI 설정 파일 (`~/.duckdbrc`)

매번 CLI를 열 때마다 동일한 설정을 반복하는 것은 번거롭다. `~/.duckdbrc` 파일에 원하는 설정을 저장해두면, DuckDB CLI가 시작할 때마다 이 파일을 자동으로 읽어 설정을 적용한다.

```sql
-- ~/.duckdbrc
.mode duckbox
SET threads = 8;
SET memory_limit = '16GB';
```

> **팁:** `SET threads`는 DuckDB가 쿼리 실행에 사용할 CPU 코어 수를 지정한다. 기본값은 시스템의 전체 코어 수이지만, 다른 작업과 병행할 때는 이 값을 낮추면 시스템 자원 경쟁을 줄일 수 있다.

---

### CLI 종료

| 단축키 / 명령 | 설명 |
|---|---|
| `Ctrl + D` | 종료 (Unix/macOS) |
| `Ctrl + C` | 종료 (Windows) |
| `.exit` | 명령으로 종료 |

영속 모드(`duckdb my_database.duckdb`)로 실행 중이었다면 종료 시 자동으로 체크포인트(checkpoint)가 수행되어 데이터가 파일에 저장된다.
