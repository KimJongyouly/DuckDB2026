# 제16장: 데이터베이스 연동

DuckDB는 분석 데이터베이스지만 기존 OLTP 데이터베이스와 단절되어 있지 않다. PostgreSQL, MySQL, SQLite에 직접 연결해 원격 테이블을 로컬 테이블처럼 쿼리하거나, 데이터를 DuckDB로 가져와 고속 분석을 수행하는 패턴이 실무에서 매우 흔하다. 이 장에서는 각 데이터베이스와의 연동 방법과 실전 패턴을 정리한다.

DuckDB가 외부 데이터베이스와 연결할 때 사용하는 핵심 메커니즘은 `ATTACH`다. `ATTACH`는 외부 데이터베이스를 DuckDB의 논리적 데이터베이스로 등록해, 연결된 테이블을 마치 로컬 테이블처럼 `db명.스키마.테이블` 형식으로 참조할 수 있게 한다. 연결이 끝나면 `DETACH`로 깔끔하게 해제한다.

```sql
-- PostgreSQL 연결 예시
ATTACH 'postgresql://postgres:secret@localhost:5432/mydb'
AS pg (TYPE POSTGRES, READ_ONLY);

-- 원격 테이블을 로컬처럼 쿼리
SELECT department, COUNT(*), AVG(salary)
FROM pg.public.employees
GROUP BY department;

-- 연결 해제
DETACH pg;
```

## 주요 학습 내용

- ATTACH는 외부 DB를 DuckDB 세션에 붙여 넣는 명령으로, 원격 테이블을 `db_name.schema.table` 형태로 로컬처럼 참조할 수 있다.
- Predicate/Projection Pushdown이 성능의 핵심이다. DuckDB가 WHERE 조건과 열 선택을 원격 DB에서 먼저 처리하도록 자동 최적화해 네트워크 전송량을 줄인다.
- 연결 문자열의 비밀번호 평문 노출은 운영 환경에서 반드시 피해야 한다. `CREATE SECRET`이나 환경 변수로 자격증명을 분리 관리하고, 분석 목적이라면 `READ_ONLY` 옵션을 습관적으로 명시한다.
