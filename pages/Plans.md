# 강력한분석디비.DuckDB — 원고 개선 Plans.md

작성일: 2026-03-20

목표: 01~24장 전체 챕터를 다음 세 가지 기준으로 개선한다.
1. **문맥 연결**: 섹션 간 어색한 전환, 갑자기 등장하는 개념, 앞뒤가 맞지 않는 설명을 자연스럽게 다듬는다.
2. **친절한 어조**: 독자가 처음 접하는 개념에는 "왜 이게 필요한지"를 먼저 짚어주고, 코드 전후에 충분한 설명을 붙인다.
3. **장 마무리 정리**: 각 장 맨 끝에 "이 장에서 배운 것"을 산문+핵심 포인트 형태로 요약하는 섹션을 추가한다. 기존의 단순 표 정리와는 별도로, 다음 장으로 자연스럽게 넘어가는 연결 문장도 포함한다.

## 개선 공통 지침

각 태스크 실행 시 아래 기준을 일관되게 적용한다.

### 문맥 연결 체크포인트
- 새로운 개념이 처음 등장할 때: 1~2문장으로 "왜 이 개념이 필요한가" 선행 설명
- 섹션 구분선(`---`) 전후: 자연스러운 전환 문장 삽입
- 코드 블록 앞: "다음 예시는 ~~를 보여준다" 식의 도입 문장 유지

### 친절한 어조 기준
- 독자를 "여러분"으로 부르지 않고, 현재형·능동형 문장으로 직접 설명
- "~하면 된다" 보다 "~하면 DuckDB가 ~~를 처리한다" 식으로 동작 원리 포함
- 주의사항·팁은 `> **팁:**` 또는 `> **주의:**` 블록쿼트로 강조

### 장 마무리 섹션 형식
```markdown
---

## 마치며

(3~5문장의 산문: 이 장에서 다룬 핵심 내용을 자연스럽게 요약)

**이 장의 핵심 포인트**

- 포인트 1
- 포인트 2
- 포인트 3
- (필요 시 추가)

(1문장: 다음 장과의 연결 — "다음 장에서는 ~~를 살펴본다.")
```
