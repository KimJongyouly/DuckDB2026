# 제2장: DuckDB에 연결하기

DuckDB를 사용하려면 가장 먼저 **연결(connection)** 을 만들어야 한다. 연결은 DuckDB 엔진에 접근하는 진입점이다. 쿼리를 실행하고, 테이블을 만들고, 데이터를 읽고 쓰는 모든 작업은 연결을 통해 이루어진다. 다른 데이터베이스와 달리 DuckDB의 연결은 서버 프로세스나 네트워크 설정 없이 라이브러리 함수 호출 한 번으로 만들어지기 때문에, 진입 장벽이 매우 낮다.

이 장에서는 연결의 개념부터 시작해 두 가지 핵심 모드—영구 모드와 인메모리 모드—, 대용량 처리를 가능하게 하는 디스크 스필링, 그리고 동시성 모델까지 차례로 살펴본다.

```python
import duckdb

# 영구 모드
con = duckdb.connect("my_database.duckdb")

# 인메모리 모드
con = duckdb.connect()
```

---

## 주요 학습 내용

- `:memory:`는 빠르고 가볍지만 세션이 끝나면 데이터가 사라진다. 영속화가 필요하다면 반드시 파일 경로를 지정해야 한다.
- DuckDB는 단일 쓰기 연결 제약이 있다. 동시에 여러 프로세스가 같은 파일을 쓰려 하면 잠금 오류가 발생하므로, 읽기 전용 연결(`read_only=True`)로 동시 읽기를 허용하는 패턴을 활용한다.
- `ATTACH` 명령으로 여러 DuckDB 파일 또는 외부 DB를 단일 세션에서 연결해 크로스 DB 쿼리를 실행할 수 있다.
