# 제13장: CLI 심화

DuckDB CLI는 터미널에서 데이터를 탐색하고 쿼리를 실행하는 가장 빠른 방법이다. 단순한 인터랙티브 셸을 넘어 배치 스크립트, 파이프라인 자동화, CI 데이터 검증까지 폭넓게 활용할 수 있다. 이 장에서는 CLI의 숨겨진 옵션과 실전 자동화 패턴을 체계적으로 정리한다.

앞서 살펴본 DuckDB의 SQL 기능과 확장들은 결국 어떤 방식으로든 실행 환경이 필요하다. Python 라이브러리나 JDBC 드라이버를 쓰지 않아도, CLI 하나만으로 데이터 수집부터 변환, 검증, 리포트 생성까지 전 과정을 자동화할 수 있다. 이 장에서 소개하는 패턴들을 익히면 DuckDB를 단순한 분석 도구가 아닌 경량 데이터 파이프라인 엔진으로 활용할 수 있다.

```bash
#!/usr/bin/env bash
# 기본 ETL 파이프라인 예시
DB="analytics.duckdb"

duckdb "$DB" << 'EOF'
    SELECT
        DATE_TRUNC('month', order_date) AS month,
        region,
        COUNT(*) AS orders,
        SUM(amount) AS revenue
    FROM sales
    WHERE order_date >= CURRENT_DATE - INTERVAL '3 months'
    GROUP BY month, region
    ORDER BY month DESC, revenue DESC;
EOF
```

## 주요 학습 내용

- Heredoc(`duckdb db.duckdb << 'SQL' ... SQL`)을 활용하면 여러 SQL 문을 하나의 셸 명령으로 실행할 수 있다. 변수 주입 시 큰따옴표 heredoc을 사용해야 `${VAR}` 치환이 적용된다.
- `-csv -noheader` 플래그로 출력을 머신 친화적 포맷으로 바꾸면 셸 변수 캡처, awk 파이프라인 연동이 쉬워진다.
- `ERROR()` 함수는 조건 실패 시 DuckDB가 비정상 종료(exit code 1)하게 만든다. CI/CD 파이프라인에서 데이터 품질 게이트로 활용할 수 있다.
