# 제15장: 핵심 확장 활용

DuckDB의 확장(extension)은 코어 엔진을 건드리지 않고 기능을 동적으로 추가하는 플러그인 시스템이다. S3 파일 직접 쿼리, 공간 데이터 분석, 전문 검색, Excel 읽기까지 — 필요한 기능만 골라 설치하면 DuckDB가 범용 분석 플랫폼으로 확장된다. 이 장에서는 실무에서 가장 자주 쓰이는 핵심 확장을 중심으로 설치부터 실전 패턴까지 다룬다.

DuckDB가 기본으로 제공하는 기능만으로도 로컬 파일 분석은 충분하지만, 클라우드 스토리지 연결·지리 정보 분석·전문 검색 같은 도메인 특화 기능은 확장을 통해 on-demand로 활성화한다. 이 방식은 불필요한 기능을 로드하지 않아 엔진을 가볍게 유지하면서도, 필요할 때 즉시 기능을 추가할 수 있다는 장점을 준다.

```sql
-- 자동 설치 · 자동 로드 설정 (권장)
SET autoinstall_known_extensions = true;
SET autoload_known_extensions = true;
-- 이후 알려진 확장은 사용 시 자동 설치·로드
```

## 주요 학습 내용

- `httpfs` 확장은 `SET s3_region`과 `CREATE SECRET`으로 인증을 설정한 뒤 S3 URL을 일반 파일 경로처럼 사용할 수 있게 해준다. 원격 Parquet 파일의 Footer만 먼저 읽어 필요한 Row Group만 내려받는 최적화가 자동으로 적용된다.
- `spatial` 확장은 PostGIS와 유사한 공간 함수를 제공한다. `ST_Distance`, `ST_Contains`, `ST_Intersects`로 지리 분석을 SQL 안에서 처리할 수 있다.
- `CREATE SECRET`으로 S3/Azure 자격증명을 객체화하면 연결 문자열에 키를 평문으로 노출하지 않아도 된다. `PERSISTENT` 옵션으로 세션을 넘어 재사용도 가능하다.
