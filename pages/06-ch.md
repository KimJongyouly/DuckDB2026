# 제6장: DDL과 DML

DDL(Data Definition Language)은 데이터베이스 구조를 정의하고, DML(Data Manipulation Language)은 데이터를 조작한다. 두 범주는 역할이 완전히 다르다. DDL은 "어디에 무엇을 담을지"를 결정하는 그릇을 만드는 작업이고, DML은 그 그릇에 데이터를 넣고 고치고 꺼내는 작업이다. 이 구분을 이해하면 SQL 오류 메시지를 해석하거나 권한을 설계할 때 훨씬 명확하게 판단할 수 있다.

DuckDB는 표준 SQL DDL/DML을 충실히 지원하면서도 분석 데이터베이스답게 대량 삽입, UPSERT, RETURNING 절 같은 편의 기능을 갖추고 있다. 이 장에서는 테이블 생성부터 트랜잭션 관리까지 데이터베이스 운영의 전 과정을 살펴본다.

```sql
-- UPSERT: 충돌 시 누적 합산 예시
INSERT INTO daily_stats (date, region, visits)
VALUES ('2024-03-15', 'Seoul', 1500)
ON CONFLICT (date, region) DO UPDATE
SET visits = daily_stats.visits + EXCLUDED.visits;
```

---

## 주요 학습 내용

- DDL은 구조(테이블·뷰·인덱스)를 정의하고, DML은 그 안의 데이터를 조작한다. 둘의 역할을 구분하면 SQL 명령어를 체계적으로 이해할 수 있다.
- UPSERT(`ON CONFLICT DO UPDATE`)와 RETURNING 절은 실무 데이터 파이프라인에서 코드를 크게 단순화한다. 특히 집계 누적이나 멱등성 보장이 필요한 상황에서 강력하다.
- `DROP TABLE`과 `TRUNCATE`처럼 데이터를 영구 삭제하는 명령은 항상 신중하게 사용해야 하며, 트랜잭션으로 감싸거나 미리 백업해두는 습관이 중요하다.
