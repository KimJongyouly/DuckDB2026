
### 컨텍스트 매니저

연결을 열었으면 닫아야 한다. Python의 `with` 문을 사용하면 블록이 끝날 때 DuckDB가 자동으로 `close()`를 호출하고 체크포인트를 수행하므로, 명시적으로 `close()`를 부르지 않아도 데이터가 안전하게 기록된다.

```python
# with 블록: 종료 시 자동으로 close()
with duckdb.connect('mydb.duckdb') as con:
    result = con.execute("SELECT COUNT(*) FROM employees").fetchone()
    print(result[0])
# 블록 탈출 시 자동 close + checkpoint
```

> **팁:** 단발성 스크립트나 함수 단위 작업에서는 `with` 문을 쓰는 것이 가장 안전하다. 반면 Jupyter Notebook처럼 세션 내내 연결을 유지해야 하는 환경에서는 노트북 상단에서 `con = duckdb.connect()`로 열고, 커널 재시작 시 자동으로 닫히도록 두는 패턴이 일반적이다.

### 기본 쿼리 실행

```python
import duckdb

con = duckdb.connect()
con.execute("CREATE TABLE t (id INTEGER, name VARCHAR)")
con.execute("INSERT INTO t VALUES (1, 'Alice'), (2, 'Bob')")

# fetchone: 첫 번째 행 (튜플)
row = con.execute("SELECT * FROM t WHERE id = 1").fetchone()
print(row)      # (1, 'Alice')

# fetchall: 전체 행 (리스트 of 튜플)
rows = con.execute("SELECT * FROM t").fetchall()
print(rows)     # [(1, 'Alice'), (2, 'Bob')]

# fetchmany: N행씩
rows = con.execute("SELECT * FROM t").fetchmany(10)

# description: 컬럼 메타데이터
cur = con.execute("SELECT * FROM t")
print(cur.description)
# [('id', ...), ('name', ...)]
```

### 파라미터화 쿼리 (SQL 인젝션 방지)

외부 입력값(사용자 입력, API 파라미터 등)을 SQL 문자열에 직접 이어 붙이면 악의적인 값이 쿼리 구조를 바꿀 수 있다. DuckDB는 `?` 플레이스홀더를 통한 파라미터화 쿼리를 지원하며, 이를 사용하면 DuckDB가 값을 안전하게 이스케이프하여 SQL 인젝션을 원천 차단한다.

```python
# ? 플레이스홀더 (위치 기반)
con.execute(
    "SELECT * FROM employees WHERE department = ? AND salary > ?",
    ['Engineering', 80000]
)

# 다중 행 파라미터화
con.executemany(
    "INSERT INTO employees (id, name, salary) VALUES (?, ?, ?)",
    [
        (1, 'Alice', 95000),
        (2, 'Bob',   82000),
        (3, 'Carol', 105000),
    ]
)

# 절대 이렇게 하지 말 것 (SQL 인젝션 위험)
dept = "Engineering"
con.execute(f"SELECT * FROM employees WHERE department = '{dept}'")  # 위험!
```

> **주의:** f-string이나 문자열 포맷팅으로 SQL을 조립하는 패턴은 간편해 보이지만, 외부 입력이 섞이는 순간 보안 취약점이 생긴다. 값은 항상 `?` 플레이스홀더로, 테이블명·컬럼명 등 식별자는 화이트리스트로 검증한 뒤 사용한다.

### 모듈 수준 기본 연결

빠른 프로토타이핑이나 스크립트 한 줄짜리 분석에서는 연결 객체를 만들지 않고 `duckdb` 모듈을 직접 호출할 수 있다. DuckDB가 내부적으로 인메모리 기본 연결을 자동 생성해 사용한다.

```python
# duckdb 모듈을 직접 사용 (내부적으로 기본 연결 사용)
import duckdb

# 연결 객체 없이 바로 사용 가능
duckdb.execute("CREATE TABLE t (x INTEGER)")
duckdb.sql("SELECT 42 AS answer").show()

# 단, 모듈 수준 연결은 인메모리·단일 연결 — 멀티스레드 환경에서 주의
```

> **주의:** 모듈 수준 기본 연결은 프로세스 전체에서 하나의 인메모리 연결을 공유한다. 멀티스레드 환경이나 여러 파일을 다루는 프로젝트에서는 `con = duckdb.connect()`로 명시적 연결 객체를 만들어 관리하는 것이 안전하다.

### 연결 설정 (SET / PRAGMA)

```python
# 세션 설정
con.execute("SET threads = 4")
con.execute("SET memory_limit = '2GB'")
con.execute("SET enable_progress_bar = true")

# 설정 조회
result = con.execute("SELECT current_setting('threads')").fetchone()
print(result[0])   # '4'

# 모든 설정 조회
settings = con.execute("SELECT * FROM duckdb_settings()").df()
```
