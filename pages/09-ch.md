# 제9장: Parquet 파일

CSV는 익숙하고 범용적이지만, 수십~수백 GB의 분석 데이터를 다룰 때는 한계가 뚜렷하다. 모든 열을 통째로 읽어야 하고, 타입 정보가 없어 매번 추론해야 하며, 압축도 외부 도구에 의존한다. Parquet는 이런 문제를 해결하기 위해 분석 워크로드 전용으로 설계된 열 지향 바이너리 파일 형식이다. 열 단위 저장, 내장 압축, 통계 메타데이터를 갖춰 대용량 분석 쿼리에서 CSV 대비 수십 배의 성능 차이를 보인다. DuckDB는 Parquet를 네이티브 수준으로 지원해, 파일을 테이블처럼 직접 쿼리할 수 있다.

```sql
-- 파일을 직접 쿼리
SELECT * FROM 'data.parquet';

-- Hive 파티션 읽기 + Partition Pruning
SELECT *
FROM read_parquet('warehouse/**/*.parquet', hive_partitioning=true)
WHERE year = 2024 AND region = 'Seoul';

-- ZSTD 압축으로 쓰기
COPY employees TO 'employees.parquet' (
    FORMAT PARQUET,
    COMPRESSION ZSTD,
    ROW_GROUP_SIZE 100000
);
```

## 주요 학습 내용

- Parquet은 열 기반(Columnar) 포맷이라 집계 쿼리에서 필요한 열만 읽는다. 같은 데이터라도 CSV보다 쿼리가 수십 배 빠를 수 있다.
- Zone Map은 각 Row Group의 min/max 통계를 Footer에 저장해, DuckDB가 WHERE 조건을 만족할 수 없는 Row Group을 통째로 건너뛰게 한다. 데이터를 정렬해 저장하면 이 효과가 극대화된다.
- `COPY ... TO 'file.parquet' (COMPRESSION ZSTD, ROW_GROUP_SIZE 100000)`으로 압축과 Row Group 크기를 직접 제어할 수 있다.
