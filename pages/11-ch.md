# 제11장: 레이크하우스 포맷

데이터 레이크는 모든 형태의 원시 데이터를 저렴하게 보관할 수 있지만, 트랜잭션 보장이나 스키마 관리 측면에서는 전통적인 데이터 웨어하우스에 미치지 못했다. 레이크하우스(Lakehouse)는 이 두 세계의 장점을 결합한 아키텍처로, 데이터 레이크의 유연성과 데이터 웨어하우스의 ACID 보장을 동시에 제공한다.

Apache Iceberg와 Delta Lake는 이 아키텍처를 구현하는 대표적인 오픈 테이블 포맷이다. 두 포맷 모두 Parquet 파일 위에 트랜잭션 로그와 메타데이터 레이어를 얹어, 파일 시스템에서도 테이블 수준의 ACID 트랜잭션·스키마 진화·타임 트래블을 가능하게 한다. DuckDB는 확장(extension)을 통해 두 포맷을 모두 읽고 쓸 수 있어, 기존 레이크하우스 인프라에 즉시 연결할 수 있다.

```sql
-- Iceberg 확장 설치 및 읽기
INSTALL iceberg;
LOAD iceberg;
SELECT * FROM iceberg_scan('path/to/iceberg_table');

-- Delta Lake 확장 설치 및 읽기
INSTALL delta;
LOAD delta;
SELECT * FROM delta_scan('path/to/delta_table');
```

## 주요 학습 내용

- Iceberg와 Delta는 "ACID 트랜잭션 + 스키마 진화 + 타임 트래블"을 Parquet 기반으로 구현한 포맷이다. DuckDB는 읽기 엔진으로서 이 포맷들과 자연스럽게 통합된다.
- Appender API(`duckdb.Appender`)는 수백만 행의 데이터를 배치로 삽입할 때 INSERT 루프보다 훨씬 빠르다. 스트리밍 데이터 적재에 적합하다.
- 적재 방법은 상황에 따라 선택한다. 파일 기반이면 `INSERT INTO t SELECT * FROM read_parquet(...)`, 스트리밍이면 Appender, 소량이면 VALUES가 적합하다.
